## 项目概述

LXC（Linux Containers）是一个成熟的、经过大规模生产环境验证的底层 Linux 容器运行时。自 2008 年以来持续开发，其核心贡献者同时参与了 Linux 内核中容器化特性的实现。

### 核心特性

- **系统容器**：提供接近虚拟机的隔离环境，但没有独立内核和硬件模拟的开销
- **非特权容器**：利用 user namespace 实现完全无特权运行
- **强向后兼容**：自 1.0.0 版本以来 API 从未破坏
- **广泛架构支持**：x86_64, arm64, ppc64le, riscv64, s390x, loongarch64 等
- **多 C 库支持**：glibc, musl

### 技术基础

LXC 通过组合以下内核特性实现容器隔离：
- **Namespace**：user, mount, pid, uts, ipc, net, cgroup, time
- **Cgroup**：资源限制与控制（v1/v2）
- **LSM**：AppArmor / SELinux 强制访问控制
- **Seccomp**：系统调用过滤
- **Capabilities**：细粒度权限控制

### 构建方式

```bash
meson setup -Dprefix=/usr build
meson compile -C build
```

## 源码目录结构

```
src/
├── include/              # 兼容性头文件（musl/缺失函数补全）
├── lxc/
│   ├── lxccontainer.c/h  # 核心容器 API
│   ├── start.c/h         # 容器启动引擎
│   ├── conf.c/h          # 配置数据结构
│   ├── confile.c/h       # 配置文件解析
│   ├── state.c/h         # 状态机
│   ├── attach.c/h        # 容器附着（lxc-attach）
│   ├── network.c/h       # 网络管理
│   ├── namespace.c/h     # 命名空间
│   ├── cgroups/           # Cgroup 子系统
│   │   ├── cgroup.c/h
│   │   ├── cgfsng.c       # 主 cgroup 驱动
│   │   └── cgroup2_devices.c/h  # BPF 设备控制
│   ├── storage/           # 存储后端
│   │   ├── storage.c/h    # 抽象层
│   │   ├── dir.c/btrfs.c/lvm.c/overlay.c/zfs.c/loop.c/...
│   ├── lsm/               # Linux 安全模块
│   │   ├── lsm.c/h        # 抽象层
│   │   ├── apparmor.c/selinux.c/nop.c
│   ├── tools/             # CLI 工具
│   │   ├── arguments.c/h  # 参数解析框架
│   │   ├── lxc_start.c / lxc_stop.c / lxc_attach.c / ...
│   ├── cmd/               # 系统辅助程序
│   │   ├── lxc_init.c / lxc_monitord.c / lxc_user_nic.c / ...
│   ├── mainloop.c/h       # 事件循环（epoll/io_uring）
│   ├── commands.c/h       # IPC 命令通道
│   ├── log.c/h            # 日志系统
│   ├── terminal.c/h       # 终端/PTY 管理
│   ├── seccomp.c          # Seccomp 过滤器
│   ├── criu.c/h           # CRIU 检查点/恢复
│   └── ...                # 其他工具模块
└── tests/                 # 测试套件
```
