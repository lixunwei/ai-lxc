# 存储子系统深度分析：抽象层与分发机制

## 1. 概述

LXC 的存储子系统位于 `src/lxc/storage/`，通过函数指针表实现**后端无关**的存储抽象。支持 8 种后端：dir、btrfs、overlay、loop、zfs、lvm、nbd、rbd。

核心设计模式：
- `struct lxc_storage_ops` — 后端操作函数指针表（类似 VFS 的 file_operations）
- `struct lxc_storage` — 存储实例状态
- `bdevs[]` — 全局后端注册表，按名称映射到 ops

---

## 2. 核心数据结构

### 2.1 struct lxc_storage_ops（后端操作表）

```c
// src/lxc/storage/storage.h:48-69
struct lxc_storage_ops {
    // 检测路径是否属于此后端
    bool (*detect)(const char *path);
    // 挂载存储到 rootfs mount point
    int (*mount)(struct lxc_storage *bdev);
    // 卸载
    int (*umount)(struct lxc_storage *bdev);
    // 销毁存储（删除 subvolume/LV/文件等）
    int (*destroy)(struct lxc_storage *bdev);
    // 创建新存储
    int (*create)(struct lxc_storage *bdev, const char *dest,
                  const char *name, struct bdev_specs *specs);
    // 为克隆准备路径
    int (*clone_paths)(struct lxc_storage *orig, struct lxc_storage *new,
                       const char *oldname, const char *cname,
                       const char *oldpath, const char *newpath,
                       int snap, uint64_t newsize,
                       struct lxc_conf *conf);
    // 全量复制
    int (*copy)(struct lxc_conf *conf, struct lxc_storage *orig,
                struct lxc_storage *new, uint64_t newsize);
    // 快照
    int (*snapshot)(struct lxc_conf *conf, struct lxc_storage *orig,
                    struct lxc_storage *new, uint64_t newsize);
    // 能力标志
    bool can_snapshot;   // 是否支持快照克隆
    bool can_backup;     // 是否支持备份
};
```

### 2.2 struct lxc_storage（存储实例）

```c
// src/lxc/storage/storage.h:76-89
struct lxc_storage {
    const struct lxc_storage_ops *ops;  // 操作函数表
    const char *type;                    // 后端类型名
    char *src;                           // 源路径 (如 "dir:/var/lib/lxc/ct/rootfs")
    char *dest;                          // 挂载点
    char *mntopts;                       // 挂载选项
    int lofd;                            // loop 设备 fd
    int nbd_idx;                         // NBD 设备索引
    uint64_t flags;                      // 挂载标志
    struct lxc_rootfs *rootfs;           // 关联的 rootfs 结构
};
```

### 2.3 bdevs[] 后端注册表

```c
// src/lxc/storage/storage.c:165-180
static const struct bdev_type bdevs[] = {
    { .name = "dir",      .ops = &dir_ops      },

    { .name = "overlayfs",.ops = &overlay_ops  },  // 兼容旧名
    { .name = "loop",     .ops = &loop_ops     },
    { .name = "zfs",      .ops = &zfs_ops      },
    { .name = "lvm",      .ops = &lvm_ops      },
    { .name = "nbd",      .ops = &nbd_ops      },
    { .name = "rbd",      .ops = &rbd_ops      },
};
```

## 4. 后端选择与分发

### 4.1 选择策略

```
storage_get(type, src):
    if type is specified:
        exact match in bdevs[] by name
    else:
        for each bdev in bdevs[]:
            if bdev.ops->detect(src):
                return this backend
        fallback to "dir"
```

### 4.2 创建时的选择

```
storage_create(dest, type, name, specs, conf):
    if type == "best":
        try btrfs -> zfs -> lvm -> dir (按优先级)
    elif type contains ',':
        split by comma, try each in order
    else:
        exact match by type name
```

### 4.3 detect 函数示例

```c
// dir: 路径存在且是目录
bool dir_detect(const char *path) {
    return starts_with("dir:") || is_directory(path);
}

// btrfs: 检查文件系统类型
bool btrfs_detect(const char *path) {
    return is_btrfs_fs(path);  // statfs() check
}

// lvm: 路径匹配 /dev/<vg>/<lv> 模式
bool lvm_detect(const char *path) {
    return starts_with("lvm:") || starts_with("/dev/");
}
```

## 5. 容器创建流程（storage 视角）

```
lxc-create -n myct -t download -B btrfs
    |
    v
lxccontainer.c: c->create()
    |
    v
do_storage_create(dest, "btrfs", "myct", specs, conf)
    |
    v
storage_create():
    +-- allocate struct lxc_storage
    +-- find "btrfs" in bdevs[]
    +-- install btrfs_ops
    +-- call btrfs_ops.create(bdev, dest, name, specs)
    |       +-- BTRFS_IOC_SUBVOL_CREATE
    |       +-- set bdev->src = "btrfs:/path/to/subvol"
    +-- set conf->rootfs.path = bdev->src
    |
    v
mount storage -> run template -> umount
```

## 6. 容器启动流程（storage 视角）

```
__lxc_start() -> lxc_spawn() -> do_start() -> lxc_setup()
    |
    v
lxc_storage_prepare(conf):
    +-- parse conf->rootfs.path (e.g. "btrfs:/var/lib/lxc/ct/rootfs")
    +-- extract type prefix ("btrfs")
    +-- storage_get("btrfs", path)
    +-- attach/prepare backend (e.g. loop: open loop device)
    +-- store in conf->rootfs.storage
    |
    v
lxc_mount_rootfs(&conf->rootfs):
    +-- if no rootfs.path: make host "/" a slave mount, use as rootfs
    +-- else:
    |     +-- access(rootfs->mount) check
    |     +-- rootfs->storage->ops->mount(storage)
    |     |       +-- (backend-specific mount logic)
    |     +-- open mounted dir for later pivot_root
    v
lxc_setup() continues: autodev, proc/sys mounts, pivot_root...
```

## 7. 克隆/快照流程

```
lxc-copy -n orig -N clone [-s for snapshot]
    |
    v
container API: c->clone()
    |
    v
determine clone type:
    if -s (snapshot) AND backend.can_snapshot:
        use snapshot path
    else:
        full copy

snapshot path (e.g. btrfs):
    +-- clone_paths(): prepare new paths
    +-- snapshot(): BTRFS_IOC_SNAP_CREATE_V2
    
full copy path:
    +-- clone_paths(): create new storage
    +-- copy(): rsync from old to new
```

## 8. 辅助工具（storage_utils.c）

| 函数 | 功能 |
|------|------|
| `detect_fs()` | 在私有 mount namespace 中探测文件系统类型 |
| `mount_unknown_fs()` | 遍历 /etc/filesystems + /proc/filesystems 尝试挂载 |
| `do_mkfs_exec_wrapper()` | fork+exec 运行 mkfs.xxx |
| `blk_getsize()` | ioctl(BLKGETSIZE64) 获取块设备大小 |
| `attach_block_device()` | NBD 设备连接 |
| `detach_block_device()` | NBD 设备断开 |
| `is_blktype()` | 判断是否为块设备后端（目前仅 lvm） |

## 9. src 路径格式

各后端使用 `type:path` 格式标识存储源：

| 后端 | src 格式 | 示例 |
|------|----------|------|
| dir | `dir:<path>` | `dir:/var/lib/lxc/ct/rootfs` |
| btrfs | `btrfs:<path>` | `btrfs:/var/lib/lxc/ct/rootfs` |
| overlay | `overlay:<lower>:<upper>` | `overlay:/lower:/upper` |
| loop | `loop:<file>` | `loop:/var/lib/lxc/ct/rootfs.img` |
| zfs | `zfs:<dataset>` | `zfs:lxc/containers/myct` |
| lvm | `lvm:<vg>/<lv>` | `lvm:vg0/ct-rootfs` |
| nbd | `nbd:<image>` | `nbd:/path/to/disk.qcow2` |
| rbd | `rbd:<pool>/<image>` | `rbd:lxc/myct-rootfs` |

## 10. 配置项

```ini
# rootfs 路径（包含后端类型前缀）
lxc.rootfs.path = dir:/var/lib/lxc/myct/rootfs
lxc.rootfs.path = btrfs:/var/lib/lxc/myct/rootfs
lxc.rootfs.path = overlay:/var/lib/lxc/myct/rootfs:/var/lib/lxc/myct/delta

# 挂载选项
lxc.rootfs.options = noatime,nodiratime

# 管理标志（内部使用）
lxc.rootfs.managed = 1
```
