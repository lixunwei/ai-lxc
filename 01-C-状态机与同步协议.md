# LXC 核心生命周期深度分析：状态机与同步协议

## 1. 概述

LXC 使用状态机管理容器生命周期，通过精心设计的同步协议协调父子进程。
本文分析状态定义、转换规则、同步消息协议、handler 结构和清理路径。

---

## 2. 容器状态定义

**位置**：`src/lxc/state.h:12-22`、`src/lxc/state.c:29-53`

### 2.1 状态枚举

```c
typedef enum {
    STOPPED,    // 已停止（初始/最终状态）
    STARTING,   // 正在启动
    RUNNING,    // 运行中
    STOPPING,   // 正在停止
    ABORTING,   // 异常中止
    FREEZING,   // 正在冻结
    FROZEN,     // 已冻结
    THAWED,     // 已解冻（瞬态，很快回到 RUNNING）
    MAX_STATE,  // 状态数量
} lxc_state_t;
```

### 2.2 状态转换图

```

                    │                          │
                    ▼                          │
 ┌─────────┐   ┌──────────┐   ┌─────────┐   ┌──────────┐

 └─────────┘   └──────────┘   └─────────┘   └──────────┘
     ▲              │              │  ▲            │
     │              │              │  │            │
     │              ▼              │  │            │
     │         ┌──────────┐       │  │            │
     │         │ ABORTING │       │  └────────────┘

     │              │             │
     │              │             ▼
     │              │        ┌──────────┐   ┌────────┐

```

### 2.3 关键转换点

| 转换 | 触发位置 | 条件 |
|------|----------|------|
| STOPPED → STARTING | `lxc_init()` (`start.c:973`) | 容器开始启动 |
| STARTING → RUNNING | `lxc_spawn()` (`start.c:2153`) | spawn 成功完成 |
| STARTING → ABORTING | `lxc_abort()` (`start.c:1224`) | 启动失败 |
| RUNNING → STOPPING | `lxc_end()` (`start.c:1111`) | 正常退出开始 |
| RUNNING → FREEZING | `cgroup_freeze()` | 冻结请求 |
| FREEZING → FROZEN | cgroup 状态回调 | 冻结完成 |
| FROZEN → THAWED | `cgroup_unfreeze()` | 解冻完成 |
| STOPPING → STOPPED | `lxc_end()` (`start.c:1170`) | 清理完成 |
| ABORTING → STOPPED | `lxc_end()` | 中止清理完成 |

### 3.2 状态客户端

```

  +-- update handler->state
  +-- walk conf->state_clients:
        for each client:

                send lxc_msg_state to client
                close client fd
                remove from list
```

## 5. 同步协议详解

**位置**：`src/lxc/sync.h:14-45`、`src/lxc/sync.c:18-157`

### 5.1 消息类型

```c
enum {
    SYNC_ERROR           = -1,  // 错误（携带 errno）
    START_SYNC_STARTUP   =  0,  // 启动信号
    START_SYNC_CONFIGURE =  1,  // 配置完成
    START_SYNC_POST_CONFIGURE = 2,  // 后配置完成
    START_SYNC_IDMAPPED_MOUNTS = 3, // ID 映射挂载就绪
    START_SYNC_CGROUP_LIMITS = 4,   // cgroup 限制就绪
    START_SYNC_FDS       =  5,  // fd 同步
    START_SYNC_READY_START = 6, // 准备启动
    START_SYNC_RESTART   =  7,  // 重启
    START_SYNC_POST_RESTART = 8, // 重启后
};
```

### 5.2 底层通信原语

```c
// Create sync socket pair
lxc_sync_init(handler)                            [sync.c:126-141]
  +-- socketpair(AF_LOCAL, SOCK_STREAM, 0, handler->sync_sock)

// Send sync message

  +-- lxc_write_nointr(fd, &sequence, sizeof(int))

// Wait for sync message
sync_wait(fd, expected_sequence)                  [sync.c:18-40]
  +-- lxc_read_nointr(fd, &msg, sizeof(int))
  +-- if msg == SYNC_ERROR:
```

## 8. ops 模式对比

**位置**：`start.h:144-147`

```c
struct lxc_operations {
    int (*start)(struct lxc_handler *, void *);
    int (*post_start)(struct lxc_handler *, void *);
};
```

### 8.1 start_ops（系统容器）

```c
// start.c:2371-2404
static struct lxc_operations start_ops = {
    .start = start_cb,         // execvp(argv[0], argv)
    .post_start = post_start_cb,
};
```

`start_cb`（`start.c:2390-2393`）直接 exec 容器的 init 程序。

### 8.2 execute_start_ops（应用容器）

```c
// execute.c:21-53
static struct lxc_operations execute_start_ops = {
    .start = execute_start_cb,  // lxc_container_init(argv)
    .post_start = NULL,
};
```

`execute_start_cb`（`execute.c:38-41`）通过 `lxc_container_init()` 启动，
该函数 fork 出子进程运行应用，父进程作为 PID 1 负责信号转发和僵尸进程收割。

### 8.3 共享引擎

两种模式共享完全相同的启动引擎（`__lxc_start → lxc_init → lxc_spawn → lxc_poll`），
仅在子进程的最终一步（`handler->ops->start()`）行为不同。
