# 03 - 网络子系统

## 1. 支持的网络类型

LXC 支持以下网络类型（`network.h:23-32`）：

| 类型 | 枚举值 | 说明 |
|------|--------|------|
| empty | `LXC_NET_EMPTY` | 仅保留 loopback 接口 |
| veth | `LXC_NET_VETH` | 虚拟以太网对（最常用） |
| macvlan | `LXC_NET_MACVLAN` | MAC 地址虚拟化 |
| ipvlan | `LXC_NET_IPVLAN` | IP 地址虚拟化 |
| phys | `LXC_NET_PHYS` | 物理网卡直通 |
| vlan | `LXC_NET_VLAN` | VLAN 子接口 |
| none | `LXC_NET_NONE` | 不创建任何接口 |

每种类型在 `network.c` 中都注册了名字和随机接口名模板（`network.c:52-65`）。

## 2. 核心数据结构

### 2.1 网络设备配置

`struct lxc_netdev`（`network.h:121-184`）：

```c
struct lxc_netdev {
    int type;               // 网络类型枚举
    int flags;              // 接口标志
    char link[IFNAMSIZ];    // 宿主侧关联接口（如桥）
    char name[IFNAMSIZ];    // 容器内接口名
    char created_name[IFNAMSIZ];   // 创建时接口名
    char transient_name[IFNAMSIZ]; // 临时接口名
    char *hwaddr;           // MAC 地址
    char *mtu;              // MTU
    union netdev_p priv;    // 类型特有数据
    struct lxc_list ipv4;   // IPv4 地址列表
    struct lxc_list ipv6;   // IPv6 地址列表
    char *ipv4_gateway;     // IPv4 网关
    char *ipv6_gateway;     // IPv6 网关
    char *upscript;         // up 钩子脚本
    char *downscript;       // down 钩子脚本
    // ...
};
```

### 2.2 类型特有数据

**veth**（`network.h:73-85`）：
```c
struct {
    char veth1[IFNAMSIZ];   // 宿主端接口名
    int ifindex;            // 宿主端 ifindex
    struct lxc_list routes; // 路由
    struct lxc_list vlans;  // VLAN 配置
};
```

**macvlan**（`network.h:87-95`）：
```c
struct {
    int mode;  // bridge / vepa / private / passthru
};
```

**ipvlan**（`network.h:97-107`）：
```c
struct {
    int mode;       // l2 / l3 / l3s
    int isolation;  // bridge / private / vepa
};
```

## 3. 网络配置流程

### 3.1 分层架构

LXC 的网络配置分为两阶段：**宿主侧配置** 和 **容器侧配置**。

通过函数指针表按类型分发（`network.c:1552-1560, 1642-1650, 1785-1793`）：

```
netdev_configure_server[type]      ← 宿主侧配置
netdev_configure_container[type]   ← 容器侧配置
netdev_deconf[type]                ← 关闭/清理
```

### 3.2 veth 配置流程（最常用）

#### 宿主侧（`network.c:616-886`）

```
lxc_veth_create()
  │
  ├── 1. 创建 veth pair
  │     └── RTM_NEWLINK + IFLA_INFO_KIND="veth" + VETH_INFO_PEER
  │
  ├── 2. 设置宿主端
  │     ├── 设置 MAC 地址
  │     ├── 设置 MTU
  │     ├── 挂接到网桥（如果配置了 link）
  │     ├── 配置 VLAN
  │     ├── 配置路由
  │     └── 配置 forwarding / proxy NDP
  │
  ├── 3. 将容器端移入容器网络命名空间
  │     └── IFLA_NET_NS_FD + RTM_NEWLINK
  │
  └── 4. 执行 hook 脚本
```

#### 容器侧（`network.c:1564-1603`）

```
容器侧配置
  ├── 通过 transient_name 找到接口的 ifindex
  ├── 重命名为 name（默认 eth0, eth1, ...）
  ├── 设置 IP 地址
  ├── 设置网关
  └── 启动接口（UP）
```

### 3.3 其他网络类型

| 类型 | 宿主侧行为 | 容器侧行为 |
|------|------------|------------|
| macvlan | 在 link 上创建 macvlan 子接口，移入容器 netns | 重命名、配置 IP/网关 |
| ipvlan | 在 link 上创建 ipvlan 子接口，移入容器 netns | 重命名、配置 IP/网关 |
| phys | 将物理接口移入容器 netns | 重命名、配置 IP/网关 |
| vlan | 在 link 上创建 VLAN 子接口，移入容器 netns | 重命名、配置 IP/网关 |
| empty | 无操作（仅保留 loopback） | 配置 loopback |
| none | 无操作 | 无操作 |

## 4. Netlink 交互

### 4.1 通用 Netlink 层（nl.c/h）

`nl.c` 提供底层 netlink socket 操作（`nl.c:20-260`）：

| 函数 | 功能 |
|------|------|
| `nlmsg_alloc()` | 分配 netlink 消息 |
| `nlmsg_reserve()` | 在消息中预留空间 |
| `nla_put_*()` | 添加各类型属性 |
| `netlink_open()` / `netlink_close()` | 打开/关闭 netlink socket |
| `netlink_send()` / `netlink_rcv()` | 发送/接收消息 |
| `netlink_transaction()` | 完整事务（发送 + 等待应答） |

### 4.2 路由 Netlink 封装（rtnl.c/h）

`rtnl.c` 是对 `nl.c` 的 `NETLINK_ROUTE` 特化（`rtnl.c:17-47`）：

```c
rtnetlink_open()        // → netlink_open(NETLINK_ROUTE)
rtnetlink_close()
rtnetlink_send()
rtnetlink_rcv()
rtnetlink_transaction() // → netlink_transaction()
```

### 4.3 network.c 中的 Netlink 用例

`network.c` 直接使用 `nl.c` 构造各种 rtnetlink 消息（`network.c:123-375, 2089-2360`）：

| 消息类型 | 用途 |
|----------|------|
| `RTM_NEWLINK` | 创建接口（veth/macvlan/ipvlan/vlan）、设置属性 |
| `RTM_GETLINK` | 查询接口信息 |
| `RTM_DELLINK` | 删除接口 |
| `RTM_NEWADDR` | 添加 IP 地址 |
| `RTM_NEWROUTE` | 添加路由 |
| `RTM_NEWNEIGH` | 添加邻居条目（proxy NDP） |

## 5. 命名空间迁移

### 5.1 接口迁移

将网络接口在宿主和容器 namespace 之间迁移（`network.h:189-199`）：

| 函数 | 功能 |
|------|------|
| `lxc_netdev_move_by_index()` | 通过 ifindex 移动接口到指定 netns |
| `lxc_netdev_move_by_name()` | 通过名称移动接口 |

实现方式：使用 `IFLA_NET_NS_FD` 属性 + `RTM_NEWLINK` 消息。

### 5.2 父子进程协调

网络创建/销毁涉及父子进程间的消息传递：

| 函数 | 方向 | 功能 |
|------|------|------|
| `lxc_network_send_to_child()` | 父 → 子 | 发送网络配置到子进程 |
| `lxc_network_recv_from_parent()` | 子 ← 父 | 子进程接收网络配置 |
| `lxc_network_send_name_and_ifindex_to_parent()` | 子 → 父 | 子进程反馈接口信息 |
| `lxc_create_network()` | — | 创建所有网络设备 |
| `lxc_delete_network()` | — | 清理所有网络设备 |
| `lxc_setup_network_in_child_namespaces()` | — | 在子 namespace 中完成网络配置 |

### 5.3 网络配置时序

```
     父进程（宿主 netns）                      子进程（容器 netns）
          │                                        │
          ├── 创建 veth pair ──────────────────────►│
          ├── 配置宿主端（桥接、VLAN、路由）        │
          ├── 将容器端移入子 netns ────────────────►│
          │                                        │
          ├── send_name_and_ifindex ───────────────►│
          │                                        ├── 接收接口信息
          │                                        ├── 重命名接口
          │                                        ├── 配置 IP/网关
          │                                        └── UP 接口
          │                                        │
          │◄── 确认完成 ──────────────────────────│
          │                                        │
```
