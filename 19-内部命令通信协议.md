# 内部命令通信协议深度分析

## 1. 概述

LXC 通过抽象 Unix 域套接字实现 handler（monitor）与外部工具（lxc-info、lxc-stop、lxc-console 等）之间的通信。每个运行中的容器有一个命令套接字，支持 27 种命令，涵盖状态查询、cgroup 操作、终端分配、进程管理等。

**核心源文件**：
- `src/lxc/commands.c` — 协议实现（服务端 + 客户端）
- `src/lxc/commands.h` — 命令枚举、消息结构定义
- `src/lxc/commands_utils.c` — 套接字路径构建、状态广播工具

## 2. 套接字路径

### 2.1 路径构建

```c
// commands_utils.c:75-143  lxc_make_abstract_socket_name()

// 标准路径 (abstract socket, 无文件系统节点)
"\0lxc/<lxcpath>/<name>/command"

// 示例:
"\0lxc//var/lib/lxc/mycontainer/command"

// 路径过长时自动切换为哈希模式:
```

**Abstract socket 优势**：不占用文件系统、无需清理、随进程退出自动消失。

## 3. 消息格式

### 3.1 请求/响应结构

```c
// commands.h:59-85
struct lxc_cmd_req {
    lxc_cmd_t cmd;        // 命令类型
    int datalen;          // 附加数据长度
    const void *data;     // 附加数据指针
};

struct lxc_cmd_rsp {
    int ret;              // 返回值 (初始化为 -ENOSYS)
    int datalen;          // 响应数据长度
    void *data;           // 响应数据
};

struct lxc_cmd_rr {
    struct lxc_cmd_req req;
    struct lxc_cmd_rsp rsp;
};
```

### 3.2 通信流程

```
Client                          Server (handler)
  |                                  |
  |-- connect (abstract socket) ---->|
  |                                  |
  |-- send(req header + cred) ------>|  lxc_cmd_accept()
  |-- send(req data, optional) ----->|
  |-- send(fd, optional SCM_RIGHTS)->|
  |                                  |
  |                                  |  lxc_cmd_handler()
  |                                  |  -> lxc_cmd_process()
  |                                  |     -> cb[req.cmd](...)
  |                                  |
  |<--- send(rsp header) ------------|
  |<--- send(rsp data, optional) ----|
  |<--- send(fd, optional) ----------|
  |                                  |
  close                              close
```

### 6.4 请求接收与验证

```c
// commands.c:2027-2092  lxc_cmd_handler()
static int lxc_cmd_handler(int fd, uint32_t events, void *data, ...) {
    struct lxc_cmd_req req;

    // 1. 接收请求头
    ret = lxc_recv_nointr(fd, &req, sizeof(req), 0);

    // 2. 验证数据长度 (防止恶意请求)
    if (req.datalen > LXC_CMD_DATA_MAX)
        goto out_close;

    // 3. 接收附加数据 (如果有)
    if (req.datalen > 0) {
        reqdata = alloca(req.datalen);
        lxc_recv_nointr(fd, reqdata, req.datalen, 0);
        req.data = reqdata;
    }

    // 4. 接收凭证 (用于权限检查)
    lxc_abstract_unix_rcv_credential(fd, &cred, ...);

    // 5. 分发到具体处理函数
    lxc_cmd_process(fd, &req, handler);
}
```

## 7. 客户端实现

### 7.1 通用客户端函数

```c
// commands.c:498-545
static int lxc_cmd(const char *name, struct lxc_cmd_rr *cmd,
                   bool *stopped, const char *lxcpath, ...) {
    // 1. 连接到容器命令套接字
    int client_fd = lxc_cmd_send(name, cmd, lxcpath, ...);

    // 2. 连接失败判定 (ECONNREFUSED = 容器已停止)
    if (client_fd < 0) {
        *stopped = (errno == ECONNREFUSED);
        return -1;
    }

    // 3. 接收响应
    ret = lxc_cmd_rsp_recv(client_fd, cmd);

    // 4. 某些命令保持连接 (GET_TTY_FD, ADD_STATE_CLIENT)
    if (stay_connected)
        cmd->rsp.ret = client_fd;  // 返回 fd 给调用者
    else
        close(client_fd);

    return ret;
}
```

## 9. 状态监听机制

### 9.1 注册监听器

```c
// 客户端调用
int lxc_cmd_add_state_client(const char *name, const char *lxcpath,
                              lxc_state_t states[MAX_STATE]) {
    struct lxc_cmd_rr cmd;
    lxc_cmd_init(&cmd, LXC_CMD_ADD_STATE_CLIENT);
    cmd.req.data = states;      // 感兴趣的状态掩码
    cmd.req.datalen = sizeof(lxc_state_t) * MAX_STATE;

    lxc_cmd(name, &cmd, &stopped, lxcpath, ...);

    if (cmd.rsp.ret < MAX_STATE)
        return cmd.rsp.ret;     // 已经处于目标状态，立即返回

    return cmd.rsp.ret;         // 返回 socket fd，等待通知
}
```

### 9.2 服务端状态客户端管理

```c
// commands_utils.c:169-194
struct lxc_state_client {
    int clientfd;                      // 客户端连接 fd
    lxc_state_t states[MAX_STATE];     // 关注的状态
    struct list_head head;             // 链表节点
};

int lxc_add_state_client(int clientfd, struct lxc_handler *handler,
                          lxc_state_t states[MAX_STATE]) {
    // 如果已经处于目标状态，直接回复
    if (states[handler->state])
        return handler->state;

    // 否则注册为观察者
    struct lxc_state_client *client = malloc(...);
    client->clientfd = clientfd;
    memcpy(client->states, states, ...);
    list_add_tail(&client->head, &handler->conf->state_clients);

    return MAX_STATE;  // 表示"已注册，等待通知"
}
```

### 9.3 状态广播

```c
// commands_utils.c:196-201
void lxc_serve_state_clients(const char *name, struct lxc_handler *handler,
                              lxc_state_t state) {
    // 遍历所有注册的状态客户端
    list_for_each_entry_safe(client, &handler->conf->state_clients, head) {
        if (client->states[state]) {
            // 发送状态通知
            lxc_send_nointr(client->clientfd, &state, sizeof(state), MSG_NOSIGNAL);
            list_del(&client->head);
            close(client->clientfd);
            free(client);
        }
    }
}
```

## 11. 安全模型

1. **Abstract socket**：命名空间隔离，仅同一网络命名空间可访问
2. **SCM_CREDENTIALS**：内核验证的进程凭证，防止伪造
3. **数据长度限制**：`LXC_CMD_DATA_MAX` 防止缓冲区溢出
4. **连接即断**：除终端和状态监听外，所有命令一次性使用后关闭
5. **一次性 handler**：每个连接只处理一条命令（`oneshot_handler`）
