# LXC 基础设施深度分析（三）：终端、日志与锁

## 1. 终端子系统

### 1.1 概述

LXC 终端子系统负责在宿主与容器之间建立 PTY（伪终端）通道，转发 I/O 数据，并处理窗口大小变化。

核心文件：`src/lxc/terminal.c`（~1350 行）、`src/lxc/terminal.h`（~100 行）

**终端结构**（`terminal.h:64-97`）：

```c

    /* PTY 对 */
    int ptx;                // 主端 fd（宿主）
    int pty;                // 从端 fd（容器）

    /* 对端（宿主侧用户终端） */

    /* 日志 */

        char *path;         // 日志文件路径
        int fd;             // 日志文件 fd

    /* 环形缓冲区（用于 console log 命令） */

    /* 状态 */

};
```

### 1.3 PTY 分配

`terminal.c:953-1045`，`lxc_devpts_terminal()`：

```
lxc_devpts_terminal()
  1. 打开 /dev/ptmx（或容器 devpts 实例）
  2. unlockpt()            // 解锁从端
  3. 尝试 TIOCGPTPEER       // 直接获取从端 fd（Linux 4.13+）
```

### 1.4 宿主-容器 I/O 转发

终端数据通过事件循环在两个方向上转发：

**容器 → 宿主**（`terminal.c:326-363`）：
```
container_pty_handler(ptx_fd)
  read(ptx_fd, buf)     // 从容器 PTY 主端读取
  write(peer_fd, buf)    // 写入宿主终端
  write(log_fd, buf)     // 可选：写入日志文件
  ringbuf_write(buf)     // 可选：写入环形缓冲区
```

**宿主 → 容器**（`terminal.c:365-383`）：
```
peer_handler(peer_fd)
  read(peer_fd, buf)     // 从宿主终端读取
  write(ptx_fd, buf)     // 写入容器 PTY 主端
```

**控制台模式**（`terminal.c:1170-1223`）：
```
stdin → container PTY      // 键盘输入传到容器
container PTY → stdout     // 容器输出显示到屏幕
```

### 1.6 SIGWINCH 处理

`terminal.c:37-143`：

```c
// 窗口大小复制
lxc_terminal_winsz(srcfd, dstfd) {
    ioctl(srcfd, TIOCGWINSZ, &wsz);  // 获取源窗口大小
    ioctl(dstfd, TIOCSWINSZ, &wsz);  // 设置目标窗口大小
}

// signalfd 初始化
lxc_terminal_signal_init() {
    sigset_t mask;
    sigaddset(&mask, SIGWINCH);
    sigaddset(&mask, SIGTERM);
    sigprocmask(SIG_BLOCK, &mask, NULL);
    return signalfd(-1, &mask, SFD_CLOEXEC);
}

// signalfd 回调
lxc_terminal_signalfd_cb(sigfd) {
    read(sigfd, &siginfo);
    if (siginfo.ssi_signo == SIGWINCH):
        lxc_terminal_winsz(peer_fd, ptx_fd);  // 同步窗口大小
    if (siginfo.ssi_signo == SIGTERM):
        return LXC_MAINLOOP_CLOSE;
}
```

### 1.7 终端连接流程

```
┌──────────┐     PTY 对      ┌──────────┐
│  宿主终端  │◄──── ptx ────►│  容器 tty  │
│  /dev/tty │     (主端)      │  /dev/pts/N│
│           │                │  (从端)    │
│  peer.fd  │◄── I/O 转发 ──►│  pty fd    │
└──────────┘                └──────────┘
     ↑                            ↑
     │ SIGWINCH                   │ 容器进程
     │ 窗口大小同步                │ stdin/stdout/stderr
```

## 3. 锁子系统

### 3.1 概述

LXC 使用 **文件锁** 防止对同一容器的并发操作。

核心文件：`src/lxc/lxclock.h`（~80 行）、`src/lxc/lxclock.c`（~320 行）

### 3.2 锁结构

`lxclock.h:30-48`：

```c
struct lxc_lock {
    enum {
        LXC_LOCK_ANON_SEM,     // 匿名信号量（进程内）
        LXC_LOCK_FLOCK,        // 文件锁（跨进程）
    } type;

    union {
        sem_t *sem;            // 信号量指针
        struct {
            int fd;            // 锁文件 fd
            char *fname;       // 锁文件路径
        } f;
    } u;
};
```

```c
        // 优先使用 OFD 锁（per-open-file，非 per-process）
        struct flock fl = {
            .l_type = F_WRLCK,
            .l_whence = SEEK_SET,
        };
        fcntl(lock->u.f.fd, F_OFD_SETLKW, &fl);
```

```c
    struct flock fl = {
```

```c
    // 1. privlock — 进程内互斥（sem_wait）
```

## 5. 设计总结

1. **PTY 对双向转发**：终端子系统通过事件循环实现宿主和容器之间的全双工 I/O 转发，支持日志记录和环形缓冲。

2. **SIGWINCH 实时同步**：通过 signalfd 将窗口大小变化通知转化为事件，实时同步到容器 PTY。

3. **分类日志**：每个源文件有独立的日志分类，可以按模块精细控制日志级别。

4. **OFD 锁优先**：使用 `F_OFD_SETLK`（Open File Description lock）而非 `flock`，提供更精确的跨线程/跨进程锁语义。

5. **两级锁**：进程内 semaphore + 跨进程文件锁的双层设计，同时处理多线程和多进程并发。

6. **EINTR 安全**：所有 I/O 函数都处理 `EINTR` 中断，确保在信号频繁的环境下可靠工作。
