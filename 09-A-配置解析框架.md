# LXC 配置解析框架

## 1. 概述

LXC 的配置系统基于 **跳转表（jump table）** 模式实现，每个 `lxc.*` 配置键都映射到一组
set/get/clr 回调函数。配置文件逐行读取后，通过前缀匹配找到对应的回调进行处理。

核心文件：
| 文件 | 职责 |
|------|------|
| `src/lxc/confile.c` | 跳转表定义、所有 set/get/clr 回调 |
| `src/lxc/confile.h` | 回调函数签名、公共 API |
| `src/lxc/confile_utils.c` | 通用辅助函数（字符串/布尔/路径设置） |
| `src/lxc/conf.h` | `struct lxc_conf` —— 全部配置存储结构 |

---

## 2. 回调函数签名

定义在 `confile.h:21-34`：

```c
/* 设置配置项 */
typedef int (*config_set_cb)(const char *key, const char *value,
                             struct lxc_conf *conf, void *data);

/* 读取配置项（写入 value 缓冲区） */
typedef int (*config_get_cb)(const char *key, char *value, int inlen,
                             struct lxc_conf *conf, void *data);

/* 清除配置项 */
typedef int (*config_clr_cb)(const char *key, struct lxc_conf *conf,
                             void *data);
```

每个跳转表条目结构为：

```c
struct lxc_config_t {
    char *name;          /* 配置键名，如 "lxc.init.cmd" */
    bool strict;         /* true = 精确匹配；false = 前缀匹配 */
    config_set_cb set;
    config_get_cb get;
    config_clr_cb clr;
};
```

## 4. 配置文件解析流程

### 4.1 入口：lxc_config_read()

```
lxc_config_read(file, conf)                    [confile.c:3191]
  ├── 设置 conf->rcfile = file
  └── lxc_file_for_each_line_mmap(file, parse_line, &plc)
        对文件每一行调用 parse_line()
```

### 4.2 行解析：parse_line()

```
parse_line(buffer, plc)                         [confile.c:3085]
  ├── 跳过空行、注释（#开头）
  ├── 跳过非 lxc. 开头的行
  ├── 以 '=' 分割为 key 和 value
  ├── 去除前后空白和引号
  ├── 调用 append_unexp_config_line() 保存原始行（非 include 场景）
  └── config = lxc_get_config(key)
      config->set(key, value, conf, NULL)       [分发到具体回调]
```

### 4.3 键查找：两种模式

**精确匹配** — `lxc_get_config_exact()` (`confile.c:331-339`)：
```c
for (i = 0; i < ARRAY_SIZE(config_jump_table); i++)
    if (strequal(name, config_jump_table[i].name))
        return &config_jump_table[i];
```

**前缀匹配** — `lxc_get_config()` (`confile.c:345-376`)：
```c
for (i = 0; i < ARRAY_SIZE(config_jump_table); i++) {
    if (config_jump_table[i].strict)
        // 精确匹配
    else
        // strnequal(name, key, strlen(key)) — 前缀匹配
}
```

排序规则（`confile.c:180-189`）：更长/更具体的前缀必须排在更短/更通用的前面。例如：
- `lxc.cgroup.dir.monitor.pivot` 排在 `lxc.cgroup.dir.monitor` 之前
- `lxc.cgroup.dir.container.inner` 排在 `lxc.cgroup.dir.container` 之前

对于超长的未知子键，通过 `-E2BIG` 路径返回 `unsupported_config_key` 回调。

## 5. 前缀匹配配置键

`strict=false` 的键使用前缀匹配，允许动态子键：

| 配置键前缀 | 用途 | 示例 |
|-----------|------|------|
| `lxc.cgroup2` | cgroup v2 控制器参数 | `lxc.cgroup2.memory.max` |
| `lxc.namespace.share.` | 共享指定命名空间 | `lxc.namespace.share.net` |
| `lxc.net.` | 网络接口配置（带索引） | `lxc.net.0.type` |
| `lxc.prlimit` | 进程资源限制 | `lxc.prlimit.nofile` |
| `lxc.sysctl` | 内核 sysctl 参数 | `lxc.sysctl.net.ipv4.ip_forward` |
| `lxc.proc` | /proc 下的进程参数 | `lxc.proc.oom_score_adj` |

### 5.1 网络索引配置的分发

`lxc.net.<N>.<subkey>` 的处理：

```
set_config_jump_table_net(key, value, conf, data)
  ├── 从 key 中解析索引 N
  ├── lxc_get_netdev_by_idx(conf, N)  → 获取或创建第 N 个网络设备
  └── lxc_get_config_net(subkey)      → 在 config_jump_table_net[] 中查找
      subconfig->set(subkey, value, conf, netdev)
```

## 7. 列表型配置

部分配置键允许多次指定，每次追加到链表：

| 配置键 | 存储方式 | 数据结构 |
|--------|----------|----------|
| `lxc.cap.drop` | 追加到链表 | `struct list_head` 字符串节点 |
| `lxc.cap.keep` | 追加到链表 | `struct list_head` 字符串节点 |
| `lxc.mount.entry` | 追加到链表 | `struct list_head` 字符串节点 |
| `lxc.hook.*` | 追加到对应钩子类型的链表 | `hooks[NUM_LXC_HOOKS]` 数组 |
| `lxc.idmap` | 追加到链表 | `struct list_head` idmap 节点 |
| `lxc.environment` | 追加到链表 | `struct list_head` 字符串节点 |
| `lxc.apparmor.raw` | 追加到链表 | `struct list_head` 字符串节点 |
| `lxc.net.<N>.ipv4.address` | 追加到网络设备的 IPv4 列表 | `struct list_head` |

对于列表型配置，`clr` 回调会遍历链表释放所有节点。

## 8. lxc.include 递归包含

`lxc.include` 配置项支持递归包含其他配置文件：

```
set_config_includefiles(key, value, conf, data)
  └── lxc_config_read(value, conf)    // 递归调用主解析函数
```

在递归解析时，`parse_line()` 不会将被包含文件的行添加到 `unexpanded_config`
（通过 `plc->from_include` 标志控制）。

## 9. 配置验证与错误处理

- **未知键**：如果在跳转表中找不到匹配，返回 `unsupported_config_key` 回调，
  该回调会打印 WARN 日志但不会导致解析失败
- **超长子键**（前缀匹配场景）：通过 `-E2BIG` 检测返回 unsupported
- **值校验**：各 set 回调内部负责，通常在无效值时返回负数错误码
- **空值处理**：部分回调（如布尔型）支持空值作为默认值设置
