# LXC 网络处理全流程：从配置到容器内生效

## 1. 概述

本文档描述一个 LXC 容器从读取 `lxc.net.*` 配置到容器内网络完全可用的**端到端流程**。该流程涉及父进程（宿主）、子进程（容器 init）两个执行上下文，通过 socketpair 协调网络设备的创建、移动和配置。

### 总体阶段

```
+---------------------------------------------------------------------------+
| Phase 1: config parsing                                                   |
|   lxc.net.X.type/link/... -> struct lxc_netdev list                       |
+---------------------------------------------------------------------------+
| Phase 2: parent creates devices (host network namespace)                  |
|   netdev_configure_server[type]() -> create veth/macvlan/...              |
+---------------------------------------------------------------------------+
| Phase 3: parent moves devices into container network namespace            |
|   lxc_netdev_move_by_index() -> device enters child netns                 |
+---------------------------------------------------------------------------+
| Phase 4: parent -> child communication (pass transient_name)              |
|   lxc_network_send_to_child() / lxc_network_recv_from_parent()            |
+---------------------------------------------------------------------------+
| Phase 5: child configures devices (container network namespace)           |
|   rename -> set MAC -> config IP -> UP -> route -> gateway               |
+---------------------------------------------------------------------------+
| Phase 6: child -> parent returns final name and ifindex                  |
|   lxc_network_send_name_and_ifindex_to_parent()                           |
+---------------------------------------------------------------------------+
```

## 2. 阶段 1：配置解析

### 2.1 配置文件示例

```ini
lxc.net.0.type = veth
lxc.net.0.link = lxcbr0
lxc.net.0.flags = up
lxc.net.0.name = eth0
lxc.net.0.hwaddr = 00:16:3e:xx:xx:xx
lxc.net.0.mtu = 1500
lxc.net.0.ipv4.address = 10.0.3.100/24
lxc.net.0.ipv4.gateway = 10.0.3.1
lxc.net.0.ipv6.address = fd00::100/64
```

### 2.2 解析路径

```
confile.c: config_jump_table_net[] (25 subkeys)
  |
  v
set_config_net_type()          -> netdev->type = LXC_NET_VETH
set_config_net_link()          -> netdev->link = "lxcbr0"
set_config_net_flags()         -> netdev->flags |= IFF_UP
set_config_net_name()          -> netdev->name = "eth0"
set_config_net_hwaddr()        -> netdev->hwaddr = "00:16:3e:..."
set_config_net_mtu()           -> netdev->mtu = "1500"
set_config_net_ipv4_address()  -> append to netdev->ipv4_addresses list
set_config_net_ipv4_gateway()  -> netdev->ipv4_gateway = (struct in_addr)
set_config_net_ipv6_address()  -> append to netdev->ipv6_addresses list
```

### 2.3 数据结构

```c
// src/lxc/network.h
struct lxc_netdev {
    int type;                    // LXC_NET_VETH / LXC_NET_MACVLAN / ...
    int flags;                   // IFF_UP etc.
    int ifindex;                 // 宿主侧 ifindex（创建后填入）
    char link[IFNAMSIZ];         // 宿主侧关联设备
    char name[IFNAMSIZ];         // 容器内目标名称
    char created_name[IFNAMSIZ]; // 创建时的临时名称
    char transient_name[IFNAMSIZ]; // 移动用的过渡名称
    char *hwaddr;                // MAC 地址
    char *mtu;                   // MTU 字符串
    struct in_addr *ipv4_gateway;
    struct in6_addr *ipv6_gateway;
    struct list_head ipv4_addresses;
    struct list_head ipv6_addresses;
    union { ... } priv;          // 类型特定属性
    struct list_head head;       // 链入 conf->netdevs
};
```

所有 `lxc.net.X.*` 配置最终汇聚到 `handler->conf->netdevs` 链表中。

## 3. 阶段 2：父进程创建网络设备

### 3.1 触发位置

```c
// src/lxc/start.c: lxc_spawn() 中
if (container_uses_namespace(handler, CLONE_NEWNET)) {
    ret = lxc_create_network(handler);  // ← 在此调用
}
```

### 3.2 分发逻辑

```c
// src/lxc/network.c
int lxc_create_network(struct lxc_handler *handler) {
    if (handler->am_root) {
        lxc_create_network_priv(handler);     // 特权模式
        lxc_network_move_created_netdev_priv(handler); // 移动设备
    } else {
        lxc_create_network_unpriv(handler);   // 非特权模式
    }
}
```

### 3.3 特权模式：netdev_configure_server 函数表

```c
static netdev_configure_server_cb netdev_configure_server[LXC_NET_MAXCONFTYPE + 1] = {
    [LXC_NET_VETH]    = netdev_configure_server_veth,
    [LXC_NET_MACVLAN] = netdev_configure_server_macvlan,
    [LXC_NET_IPVLAN]  = netdev_configure_server_ipvlan,
    [LXC_NET_VLAN]    = netdev_configure_server_vlan,
    [LXC_NET_PHYS]    = netdev_configure_server_phys,
    [LXC_NET_EMPTY]   = netdev_configure_server_empty,
    [LXC_NET_NONE]    = netdev_configure_server_none,
};
```

### 3.4 veth 创建详细流程（最常用）

```
netdev_configure_server_veth():
|
+-- 1. generate two random names: veth1("vethXXXXXX") + veth2("vethXXXXXX")
|      or use user-defined lxc.net.X.veth.pair as veth1
|
+-- 2. determine MTU (priority: config value > link device MTU > default 1500)
|
+-- 3. lxc_veth_create(veth1, veth2, pid, mtu, rxqueues, txqueues)
|      v  Netlink RTM_NEWLINK message
|      kernel creates the veth pair; veth2 is created directly in child netns (via pid)
|
+-- 4. netdev->ifindex = 0  (veth2 is already in child netns, no move needed)
|      netdev->transient_name = veth2  (tell child which device name to use)
|
+-- 5. setup_private_host_hw_addr(veth1)
|      change high MAC byte on host side to 0xfe, so the bridge does not use container MAC
|
+-- 6. lxc_netdev_set_mtu(veth1, mtu)
|
+-- 7. if link exists and mode=bridge:
|      +-- disable IPv6 link-local on host-side veth
|      +-- lxc_bridge_attach(link, veth1)  // attach veth1 to bridge
|      `-- configure VLAN (OVS or native bridge)
|
+-- 8. if mode=router:
|      +-- enable IP forwarding (ip_forwarding_on)
|      `-- add NDP/ARP proxy (lxc_ip_neigh_proxy)
|
+-- 9. lxc_netdev_up(veth1)  // bring up host-side veth
|
+-- 10. setup_ipv4_routes / setup_ipv6_routes  // host-side routes
|
`-- 11. run upscript (if lxc.net.X.script.up is configured)
```

### 3.5 非特权模式

非特权容器只支持 **veth** 类型，通过调用外部程序 `lxc-user-nic` 创建：

```c
lxc_create_network_unpriv():
    // 仅允许 LXC_NET_VETH
    lxc_create_network_unpriv_exec(lxcpath, lxcname, netdev, pid, hooks_version)
    // 执行 setuid 程序 lxc-user-nic 来创建 veth 并接入网桥
```

## 4. 阶段 3：设备移动到容器 netns

### 4.1 哪些设备需要移动

```c
static int netdev_requires_move(const struct lxc_netdev *netdev) {
    if (netdev->type == LXC_NET_EMPTY || netdev->type == LXC_NET_NONE)
        return false;
    if (!netdev->ifindex)  // veth 的 peer 端已在容器 netns
        return false;
    return true;
}
```

- **veth**：peer 端（veth2）创建时已通过 `IFLA_NET_NS_PID` 直接放入容器 netns → **不需要移动**
- **macvlan/ipvlan/phys/vlan**：先在宿主创建 → 再通过 `RTM_NEWLINK + IFLA_NET_NS_PID` 移动

### 4.2 transient name 机制

为避免设备名冲突，移动时使用随机 transient 名称：

```c
lxc_network_move_created_netdev_priv():
    for each netdev in &handler->conf->netdevs:
        create_transient_name(netdev)  // generate a random name like "physXXXXXX"
        lxc_netdev_move_by_index(netdev->ifindex, pid, netdev->transient_name)
        // move the device into pid's netns and rename it to transient_name
```

## 5. 阶段 4：父子进程网络信息传递

### 5.1 父→子：传递设备名

```c
// parent process (inside lxc_spawn(), after START_SYNC_POST_CONFIGURE)
lxc_network_send_to_child(handler):
    for each netdev:
        send(data_sock[0], netdev->name, IFNAMSIZ)             // target name
        send(data_sock[0], netdev->transient_name, IFNAMSIZ)   // current name

// child process (inside lxc_setup())
lxc_network_recv_from_parent(handler):
    for each netdev:
        recv(data_sock[1], netdev->name, IFNAMSIZ)
        recv(data_sock[1], netdev->transient_name, IFNAMSIZ)
```

### 5.2 为什么需要通信

- 父进程为 veth peer 生成了随机名称 (veth2buf)
- 子进程需要知道这个名称才能找到设备并重命名

## 6. 阶段 5：子进程网络配置

### 6.1 触发位置

```c
// src/lxc/conf.c: lxc_setup() 第 4 步
if (container_uses_namespace(handler, CLONE_NEWNET)) {
    lxc_network_recv_from_parent(handler);           // 接收名称
    lxc_setup_network_in_child_namespaces(lxc_conf); // 配置网络
}
```

### 6.2 配置流程

```
lxc_setup_network_in_child_namespaces():
|
+-- first pass: handle devices with an explicit name (e.g. lxc.net.0.name = eth0)
|   `-- netdev_configure_container[type](netdev)
|       `-- __netdev_configure_container_common(netdev)
|
+-- second pass: handle devices using automatic "eth%d" naming
|   `-- same path as above
|
`-- after each device is configured, call lxc_network_setup_in_child_namespaces_common()
```

### 6.3 设备重命名（__netdev_configure_container_common）

```c
__netdev_configure_container_common(netdev):
    // 1. find ifindex by transient_name
    netdev->ifindex = if_nametoindex(netdev->transient_name);

    // 2. if no name was specified, default to "eth%d"
    if (is_empty_string(netdev->name))
        strlcpy(netdev->name, "eth%d", IFNAMSIZ);

    // 3. rename the device (Netlink RTM_NEWLINK)
    if (!strequal(netdev->transient_name, netdev->name))
        lxc_netdev_rename_by_name(netdev->transient_name, netdev->name);

    // 4. read back the actual kernel-assigned name (eth%d -> eth0)
    if_indextoname(netdev->ifindex, current_ifname);
    strlcpy(netdev->name, current_ifname, IFNAMSIZ);
```

### 6.4 IP 和路由配置（lxc_network_setup_in_child_namespaces_common）

```c
lxc_network_setup_in_child_namespaces_common(netdev):
|
+-- 1. setup_hw_addr()         set MAC address
+-- 2. setup_ipv4_addr()       add all IPv4 addresses (RTM_NEWADDR)
+-- 3. setup_ipv6_addr()       add all IPv6 addresses
+-- 4. lxc_netdev_up(name)     bring interface up (IFF_UP)
|   `-- lxc_netdev_up("lo")   also bring up loopback
+-- 5. IPv4 gateway routes:
|   +-- lxc_ipv4_gateway_add() add default route
|   `-- if gateway is outside the subnet, add a /32 device route first, then add the default route
`-- 6. IPv6 gateway routes:
    +-- lxc_ipv6_gateway_add() add default route
    `-- if gateway is outside the subnet, add a /128 device route first, then add the default route
```

## 7. 阶段 6：回传最终信息

```c
// 子进程配置完成后
lxc_network_send_name_and_ifindex_to_parent(handler):
    for each netdev:
        send(data_sock[0], netdev->name, IFNAMSIZ)        // 最终名称
        send(data_sock[0], &netdev->ifindex, sizeof(int)) // 容器内 ifindex
```

父进程接收后可用于 `lxc-info` 等工具查询容器网络状态。

## 8. 完整时序图

```
     parent process (lxc_spawn)                 child process (do_start -> lxc_setup)
     ===========================                 ==================================
          |                                               |
          |-- clone3(CLONE_NEWNET|...) -----------------> |
          |                                               |
     [START_SYNC_CONFIGURE]                               | wait for sync
          |                                               |
   lxc_create_network()                                   |
     +-- netdev_configure_server_veth()                   |
     |   `-- lxc_veth_create(veth1, veth2, pid)           |
     |       (veth2 is created directly in child netns)   |
     +-- lxc_bridge_attach(lxcbr0, veth1)                 |
     `-- lxc_netdev_up(veth1)                             |
          |                                               |
   lxc_network_move_created_netdev_priv()                 |
     `-- (for macvlan/phys types) move into child netns   |
          |                                               |
     [START_SYNC_POST_CONFIGURE] -----------------------> | signal received, continue
          |                                               |
   lxc_network_send_to_child()                            |
     `-- send (name, transient_name) pair ------------->  |
          |                                         lxc_network_recv_from_parent()
          |                                               |
          |                                               | lxc_setup_network_in_child_namespaces()
          |                                               | +-- if_nametoindex(transient_name)
          |                                               | +-- rename transient -> "eth0"
          |                                               | +-- setup_hw_addr()
          |                                               | +-- setup_ipv4_addr() / ipv6

          |                                               |
          |                                         lxc_network_send_name_and_ifindex_to_parent()
          |<--------------------------------------------- | (name, ifindex)
          |                                               |
     [later stages: cgroup/mount/...]               [later stages: execvp init]
```

## 9. 各网络类型的差异

| 类型 | 宿主侧创建 | 是否需要 move | 容器内设备来源 |
|------|------------|--------------|---------------|
| veth | 创建 pair，veth2 直接在容器 netns | ❌ | clone 时通过 IFLA_NET_NS_PID |
| macvlan | 在 link 上创建 macvlan 子接口 | ✅ | move_by_index |
| ipvlan | 在 link 上创建 ipvlan 子接口 | ✅ | move_by_index |
| phys | 直接使用物理设备 | ✅ | move_by_index |
| vlan | 在 link 上创建 VLAN 子接口 | ✅ | move_by_index |
| empty | 无操作（仅有 loopback） | ❌ | 新 netns 自动获得 lo |
| none | 不创建 network namespace | ❌ | 共享宿主网络 |

## 10. veth 的两种模式

### 10.1 Bridge 模式（默认）

```ini
lxc.net.0.type = veth
lxc.net.0.link = lxcbr0         # 接入网桥
lxc.net.0.veth.mode = bridge    # 默认值
```

- 宿主侧 veth1 接入网桥
- 禁用 veth1 的 IPv6 link-local
- 容器通过网桥通信

### 10.2 Router 模式

```ini
lxc.net.0.type = veth
lxc.net.0.link = eth0
lxc.net.0.veth.mode = router
lxc.net.0.ipv4.address = 10.0.3.100/32
lxc.net.0.ipv4.gateway = 169.254.0.1
```

- 不接入网桥，使用路由转发
- 宿主侧启用 IP 转发 (`ip_forwarding_on`)
- 添加 ARP/NDP 代理让网络可达
- 适合云环境每个容器一个路由

## 11. 网络相关的钩子脚本

| 钩子 | 时机 | 用途 |
|------|------|------|
| `lxc.net.X.script.up` | veth 宿主侧创建并 UP 后 | 添加 iptables 规则、DHCP 等 |
| `lxc.net.X.script.down` | 容器停止、veth 销毁前 | 清理规则 |
| `lxc.hook.start-host` | 容器启动后（宿主侧）| 网络后处理 |

脚本参数：
```bash
# script.up 接收: <container-name> net <type> <veth-host-name> <link>
/etc/lxc/veth-up.sh mycontainer net veth veth1234AB lxcbr0
```

## 12. 非特权容器的网络限制

| 限制 | 原因 |
|------|------|
| 仅支持 veth | Netlink 操作需要 CAP_NET_ADMIN |
| 必须使用 lxc-user-nic | setuid 程序代行特权操作 |
| 不能设置 MTU | lxc-user-nic 不传递 MTU 参数 |
| 需要 `/etc/lxc/lxc-usernet` 授权 | 控制用户可用的网桥和设备数量 |

## 13. 配置完整参考

```ini
# === 完整 veth + bridge 配置示例 ===
lxc.net.0.type = veth
lxc.net.0.link = lxcbr0
lxc.net.0.veth.pair = mycontainer-eth0    # 可选：指定宿主侧 veth 名称
lxc.net.0.veth.mode = bridge              # bridge(默认) 或 router
lxc.net.0.flags = up
lxc.net.0.name = eth0                     # 容器内设备名
lxc.net.0.hwaddr = 00:16:3e:xx:xx:xx
lxc.net.0.mtu = 1500
lxc.net.0.ipv4.address = 10.0.3.100/24
lxc.net.0.ipv4.address = 10.0.3.101/24   # 可多个 IP
lxc.net.0.ipv4.gateway = 10.0.3.1        # 或 "auto" 或 "dev"
lxc.net.0.ipv6.address = fd00::100/64
lxc.net.0.ipv6.gateway = fd00::1
lxc.net.0.l2proxy = 1                    # L2 代理（NDP/ARP）
lxc.net.0.script.up = /etc/lxc/net-up.sh
lxc.net.0.script.down = /etc/lxc/net-down.sh
lxc.net.0.veth.vlan.id = 100             # VLAN tag
```
