# 03-B 网络子系统深入分析：各网络类型实现

## 1. veth（虚拟以太网对）— 最常用的网络类型

### 1.1 宿主侧完整流程

`netdev_configure_server_veth()`（`network.c:616-886`）：

```
netdev_configure_server_veth()
  |
  +-- 1. generate ifname
  |     +-- user set pair -> use given name
  |     +-- else -> lxc_ifname_alnum_case_sensitive("vethXXXXXX")
  |
  +-- 2. create veth pair
  |     +-- lxc_veth_create(veth1, veth2, handler->pid, mtu, rxq, txq)
  |         +-- build RTM_NEWLINK msg
  |         +-- IFLA_INFO_KIND = "veth"
  |         +-- VETH_INFO_PEER sets:
  |         |   +-- peer name = veth2
  |         |   +-- MTU(optional)
  |         |   +-- IFLA_NET_NS_PID = handler->pid
  |         |       ^ create peer directly in target netns!
  |         +-- local name = veth1
  |
  +-- 3. set host-end MAC
  |     +-- setup_private_host_hw_addr(veth1)
  |         +-- SIOCGIFHWADDR read current MAC
  |         +-- set first byte to 0xfe
  |         +-- SIOCSIFHWADDR write back
  |         goal: stop bridge MAC learning
  |
  +-- 4. set MTU(optional)
  |     +-- lxc_netdev_set_mtu(veth1, mtu)
  |         +-- RTM_NEWLINK + IFLA_MTU
  |
  +-- 5. Bridge attach(mode == VETH_MODE_BRIDGE)
  |     +-- check bridge exists
  |     +-- lxc_bridge_attach(netdev->link, veth1)
  |         +-- normal bridge -> ioctl(SIOCBRADDIF)
  |         +-- OVS bridge -> ovs-vsctl add-port
  |
  +-- 6. Bridge VLAN config(optional)
  |     +-- setup_veth_native_bridge_vlan()
  |         +-- check vlan_filtering = 1
  |         +-- read default_pvid
  |         +-- lxc_bridge_vlan_del() delete default PVID
  |         +-- lxc_bridge_vlan_add() set new PVID(untagged)
  |         +-- lxc_bridge_vlan_add_tagged() add tagged VLANs
  |
  +-- 7. Router mode config(mode == VETH_MODE_ROUTER)
  |     +-- IPv4:
  |     |   +-- lxc_ip_forwarding_on(veth1, AF_INET)
  |     |   +-- lxc_ip_neigh_proxy(RTM_NEWNEIGH, AF_INET, ifindex, gateway)
  |     +-- IPv6:
  |         +-- check net.ipv6.conf.all.forwarding = 1
  |         +-- lxc_ip_forwarding_on(veth1, AF_INET6)
  |         +-- lxc_neigh_proxy_on(veth1, AF_INET6)
  |         +-- lxc_ip_neigh_proxy(RTM_NEWNEIGH, AF_INET6, ifindex, gateway)
  |
  +-- 8. host-side routes(optional)
  |     +-- setup_ipv4_routes() -> lxc_ip_route_dest(RTM_NEWROUTE, ...)
  |     +-- setup_ipv6_routes() -> lxc_ip_route_dest(RTM_NEWROUTE, ...)
  |
  +-- 9. bring up host end
  |     +-- lxc_netdev_up(veth1) -> IFF_UP
  |
  +-- 10. run upscript(optional)
        +-- run_script_argv("net", upscript, "up", {"veth", link, veth1, NULL})
```

### 1.2 veth 的关键设计要点

1. **对端直接创建到目标 netns**：通过 `IFLA_NET_NS_PID`，veth 对端在 `clone3()` 后直接创建到容器的 network namespace 中，无需额外的 `move` 操作

2. **MAC 地址防泄漏**：修改宿主端 MAC 首字节为 `0xfe`，防止 bridge 的 MAC 学习机制将容器 MAC 暴露到宿主侧

3. **两种工作模式**：
   - `VETH_MODE_BRIDGE`：宿主端挂接到网桥，容器通过桥接访问外部网络
   - `VETH_MODE_ROUTER`：宿主端作为路由器，通过 IP forwarding 和 proxy NDP/ARP 转发流量

### 1.3 宿主侧 Router 模式详解

Router 模式不使用网桥，而是通过 sysctl 和邻居代理实现连通：

```
host                               container
+-------------+                    +-------------+
|             | veth1        veth2 |             |
|  ip_forward +--------------------+ eth0        |
|  = 1        |                    | 10.0.0.2/24 |
|             |                    | gw: 10.0.0.1|
| proxy_arp   |                    |             |
| = 1         |                    +-------------+
|             |
| neigh_proxy | <- RTM_NEWNEIGH + NTF_PROXY
| 10.0.0.2    |   host answers container ARP
+-------------+
```

## 2. macvlan

### 2.1 宿主侧流程

`netdev_configure_server_macvlan()`（`network.c:888-962`）：

```
netdev_configure_server_macvlan()
  |
  +-- 1. check link if exists
  |
  +-- 2. generate temp name "mcXXXXXX"
  |
  +-- 3. lxc_macvlan_create(link, name, mode)
  |     +-- RTM_NEWLINK:
  |         +-- IFLA_IFNAME = name
  |         +-- IFLA_LINK = parent_ifindex
  |         +-- IFLA_LINKINFO:
  |             +-- IFLA_INFO_KIND = "macvlan"
  |             +-- IFLA_INFO_DATA:
  |                 +-- IFLA_MACVLAN_MODE = mode
  |
  +-- 4. record created_name / ifindex
  |
  +-- 5. set MTU(optional)
  |
  +-- 6. run upscript(optional)
```

### 2.2 支持的模式

| 模式 | 说明 |
|------|------|
| `private` | 同宿主接口的 macvlan 之间完全隔离 |
| `vepa` | 流量经外部交换机转发 |
| `bridge` | 同宿主接口的 macvlan 之间可直接通信 |
| `passthru` | 直接继承父接口，只允许一个 macvlan |

### 2.3 容器侧

走通用逻辑 `__netdev_configure_container_common()`：
- 按 `transient_name` 找到接口 → 重命名 → 配置 IP/网关/UP

## 3. ipvlan

### 3.1 宿主侧流程

`netdev_configure_server_ipvlan()`（`network.c:1038-1110`）：

```
netdev_configure_server_ipvlan()
  |
  +-- 1. check link if exists
  |
  +-- 2. generate temp name "ipXXXXXX"
  |
  +-- 3. lxc_ipvlan_create(link, name, mode, isolation)
  |     +-- RTM_NEWLINK:
  |         +-- IFLA_IFNAME = name
  |         +-- IFLA_LINK = parent_ifindex
  |         +-- IFLA_LINKINFO:
  |             +-- IFLA_INFO_KIND = "ipvlan"
  |             +-- IFLA_INFO_DATA:
  |                 +-- IFLA_IPVLAN_MODE = mode
  |                 +-- IFLA_IPVLAN_ISOLATION = isolation  (optional)
  |
  +-- 4. record created_name / ifindex
  |
  +-- 5. set MTU(optional)
  |
  +-- 6. run upscript(optional)
```

### 3.2 支持的模式与隔离

| 参数 | 可选值 | 说明 |
|------|--------|------|
| `mode` | l2 | 二层模式，支持 ARP |
| | l3 | 三层模式，不支持广播/多播 |
| | l3s | 三层模式 + 源地址验证 |
| `isolation` | bridge | 同父接口的 ipvlan 可互通（默认） |
| | private | 完全隔离 |
| | vepa | 经外部交换机转发 |

### 3.3 L2 代理

当 `netdev->l2proxy = true` 且 ipvlan mode 为 `l3s` 时：
- `lxc_setup_l2proxy()` 会开启 `proxy_ndp`
- 为容器的每个 IP 地址添加邻居代理条目

## 4. phys（物理网卡直通）

### 4.1 宿主侧流程

`netdev_configure_server_phys()`（`network.c:1443-1518`）：

```
netdev_configure_server_phys()
  |
  +-- 1. if_nametoindex(link) -> get host-side ifindex
  |
  +-- 2. save original MTU -> priv.phys_attr.mtu(for restore)
  |
  +-- 3. read and save altname list
  |
  +-- 4. clear altname(avoid move conflicts)
  |
  +-- 5. set MTU(if configured)
  |
  +-- 6. run upscript(optional)
```

### 4.2 接口迁移

物理接口通过 netlink 移动到容器 netns：

```
lxc_netdev_move_by_name(link, pid, newname)
  +-- if_nametoindex(link)
  +-- check if WLAN:
  |   +-- [yes] is_wlan(link) -> get phy name
  |   |   +-- lxc_netdev_move_wlan(phy, link, pid, newname)
  |   |       +-- run: iw phy <phy> set netns <pid>
  |   +-- [no] lxc_netdev_move_by_index(ifindex, pid, newname)
  |       +-- RTM_NEWLINK + IFLA_NET_NS_PID
  +-- return
```

> **WLAN 特殊处理**：无线网卡不能通过 netlink `IFLA_NET_NS_PID` 移动，
> 必须使用 `iw` 命令操作 phy80211 设备。

### 4.3 恢复

容器停止时，物理接口会被恢复到宿主 netns：
- `lxc_restore_phys_nics_to_netns()` 负责将接口移回
- 恢复原始 MTU 和 altname

## 6. empty 和 none

### 6.1 empty — 仅 loopback

`netdev_configure_server_empty()`（`network.c:1521-1544`）：

```
设置 ifindex = 1（lo）
接口名必须为 "lo"
只在有 upscript 时执行脚本
容器侧：无额外操作
```

用途：容器不需要外部网络，但需要 loopback（大多数应用需要）。

### 6.2 none — 完全无网络

`netdev_configure_server_none()`（`network.c:1546-1550`）：

```
设置 ifindex = 0
容器侧：无操作
```

用途：完全不需要网络的容器（如计算型容器）。

## 7. 容器侧通用配置

### 7.1 通用重命名逻辑

`__netdev_configure_container_common()`（`network.c:1564-1603`）：

```
__netdev_configure_container_common()
  |
  +-- 1. if_nametoindex(transient_name) -> get ifindex
  |
  +-- 2. pick final ifname
  |     +-- name empty -> auto name "eth0", "eth1", ...
  |     +-- name non-empty -> use configured name
  |
  +-- 3. if transient_name != name:
  |     +-- lxc_netdev_rename_by_name(transient_name, name)
  |         +-- RTM_NEWLINK + IFLA_IFNAME
  |
  +-- 4. if_indextoname() update netdev->name
  |
  +-- 5. clear transient_name
```

### 7.2 IP 和网关配置

在容器 namespace 中统一执行（`lxc_setup_network_in_child_namespaces_common()`）：

```
lxc_setup_network_in_child_namespaces_common()
  |
  +-- 1. set MAC addr(if configured hwaddr)
  |
  +-- 2. setup_ipv4_addr()
  |     +-- walk ipv4_addresses -> lxc_ipv4_addr_add()
  |         +-- RTM_NEWADDR + IFA_LOCAL + IFA_ADDRESS + IFA_BROADCAST
  |
  +-- 3. setup_ipv6_addr()
  |     +-- walk ipv6_addresses -> lxc_ipv6_addr_add()
  |         +-- RTM_NEWADDR + IFA_LOCAL + IFA_ADDRESS
  |
  +-- 4. IPv4 gateway
  |     +-- gateway_dev -> lxc_ipv4_gateway_add(ifindex, NULL)
  |     +-- gateway -> lxc_ipv4_gateway_add(ifindex, gateway)
  |         +-- on failure:add /32 host route, then retry
  |
  +-- 5. IPv6 gateway(same as IPv4)
  |     +-- on failure:add /128 host route, then retry
  |
  +-- 6. bring up if
  |     +-- lxc_netdev_up(name) -> IFF_UP
  |
  +-- 7. bring up loopback
        +-- lxc_netdev_up("lo") -> IFF_UP
```

## 8. 父子进程网络协调

### 8.1 协调时序

```
parent(host netns)                        child(container netns)
     |                                          |
     +-- lxc_create_network()                   |
     |   +-- for each netdev:                     |
     |       netdev_configure_server[type]()     |
     |                                          |
     +-- lxc_network_send_to_child() ---------->|
     |   +-- send each netdev                  |
     |       name + transient_name               |
     |                                          |
     |                              lxc_network_recv_from_parent()
     |                                | recv name + transient_name
     |                                |
     |                              lxc_setup_network_in_child_namespaces()
     |                                | +-- __netdev_configure_container_common()
     |                                | |   +-- rename if
     |                                | +-- config IP/gateway
     |                                | +-- UP if
     |                                |
     |<------ lxc_network_send_name_and_ifindex_to_parent()
     |        +-- send final name + ifindex
     |
     +-- lxc_network_recv_name_and_ifindex_from_child()
     |   +-- save into netdev struct
     |
     +-- [continue later startup]
```

### 8.2 关闭流程

```
lxc_delete_network()
  +-- for each netdev:
      +-- netdev_deconf[type]()
      |   +-- veth: host end auto-deleted by kernel(when peer is deleted)
      |   +-- macvlan/ipvlan/vlan: lxc_netdev_delete_by_index()
      |   +-- phys: restore to host netns + restore MTU/altname
      |
      +-- run downscript(if configured)
          +-- run_script_argv("net", downscript, "down", {type, link, name, NULL})
```

## 9. 各类型对照总表

| 类型 | 创建方式 | 移入 netns | 桥接 | netlink 消息 | 快照/清理 |
|------|----------|------------|------|-------------|-----------|
| veth | `RTM_NEWLINK` + VETH_INFO_PEER | 创建时直接在目标 netns | ioctl(SIOCBRADDIF) 或 OVS | `RTM_NEWLINK` | 删除本端，对端自动消失 |
| macvlan | `RTM_NEWLINK` + IFLA_MACVLAN_MODE | `IFLA_NET_NS_PID` | 无需桥 | `RTM_NEWLINK` | `RTM_DELLINK` |
| ipvlan | `RTM_NEWLINK` + IFLA_IPVLAN_MODE | `IFLA_NET_NS_PID` | 无需桥 | `RTM_NEWLINK` | `RTM_DELLINK` |
| phys | 无需创建 | `IFLA_NET_NS_PID` 或 `iw phy` | 无 | `RTM_NEWLINK`(移动) | 恢复回宿主 netns |
| vlan | `RTM_NEWLINK` + IFLA_VLAN_ID | `IFLA_NET_NS_PID` | 无需桥 | `RTM_NEWLINK` | `RTM_DELLINK` |
| empty | 无 | 无 | 无 | 无 | 无 |
| none | 无 | 无 | 无 | 无 | 无 |
