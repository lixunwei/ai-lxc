# LXC 其他关键配置项详解

## 1. 根文件系统配置（lxc.rootfs.*）

### 1.1 配置项

| 配置键 | 存储字段 | 说明 |
|--------|----------|------|
| `lxc.rootfs.path` | `conf->rootfs.path` | rootfs 路径（含存储后端前缀） |
| `lxc.rootfs.mount` | `conf->rootfs.mount` | rootfs 挂载点 |
| `lxc.rootfs.options` | `conf->rootfs.mnt_opts` | 挂载选项（如 ro, noatime） |
| `lxc.rootfs.managed` | `conf->rootfs.managed` | 是否由 LXC 管理生命周期 |

### 1.2 rootfs.path 格式

支持存储后端前缀：

```ini
lxc.rootfs.path = dir:/var/lib/lxc/mycontainer/rootfs
lxc.rootfs.path = overlayfs:/lower:/upper
lxc.rootfs.path = loop:/path/to/image.img
lxc.rootfs.path = btrfs:/var/lib/lxc/mycontainer/rootfs
lxc.rootfs.path = zfs:tank/lxc/mycontainer
```

**解析**（`confile.c:2786-2855`）：`set_config_rootfs_path()` 调用
`set_config_string_item()`，同时提取 `__bdev_type`。

### 1.3 运行时应用

rootfs 的设置和 pivot 在 `conf.c:3830-3918` 完成：

```
lxc_setup()
  ├── lxc_rootfs_init()      — 初始化 rootfs 结构
  ├── lxc_rootfs_prepare()   — 根据存储后端准备（挂载 overlay 等）
  ├── lxc_rootfs_mount()     — 挂载 rootfs 到临时目录
  └── lxc_rootfs_pivot()     — pivot_root() 切换根文件系统
```

## 2. 环境变量配置（lxc.environment*）

### 2.1 三种环境变量

| 配置键 | 作用范围 | 存储字段 |
|--------|----------|----------|
| `lxc.environment` | init 进程继承 | `conf->environment` |
| `lxc.environment.runtime` | 仅运行时（不持久化） | `conf->environment_runtime` |
| `lxc.environment.hooks` | 仅钩子脚本可见 | `conf->environment_hooks` |

### 2.2 格式

```ini
lxc.environment = KEY=VALUE
lxc.environment = PATH=/usr/bin:/usr/sbin
lxc.environment = LANG=en_US.UTF-8
```

### 2.3 应用时机

```
start.c:
  ├── 容器启动前：注入 environment + environment_runtime 到子进程
  │   （start.c:1452-1458, 1557-1563）
  └── 钩子执行：注入 environment_hooks 到钩子进程环境
```

## 3. 钩子脚本配置（lxc.hook.*）

### 3.1 钩子类型

| 配置键 | 执行时机 | 执行上下文 |
|--------|----------|-----------|
| `lxc.hook.pre-start` | 容器启动前 | 宿主机命名空间 |
| `lxc.hook.pre-mount` | rootfs 挂载前 | 容器命名空间 |
| `lxc.hook.mount` | rootfs 挂载后 | 容器命名空间 |
| `lxc.hook.autodev` | /dev 自动填充后 | 容器命名空间 |
| `lxc.hook.start` | 容器 init 启动前 | 容器命名空间（PID 1 内） |
| `lxc.hook.start-host` | 容器 init 启动前 | 宿主机命名空间 |
| `lxc.hook.stop` | 容器停止时 | 宿主机命名空间 |
| `lxc.hook.post-stop` | 容器完全停止后 | 宿主机命名空间 |
| `lxc.hook.clone` | 容器克隆后 | 宿主机命名空间 |
| `lxc.hook.destroy` | 容器销毁前 | 宿主机命名空间 |

### 3.2 存储

```c
// conf.h:465-468
struct list_head hooks[NUM_LXC_HOOKS];  // 每个钩子类型一个链表
int hooks_version;                       // 钩子 API 版本（0 或 1）
```

### 3.3 钩子版本

| 版本 | 传参方式 |
|------|----------|
| 0 | 命令行参数：`script container_name section hook_type` |
| 1 | 环境变量：`LXC_HOOK_TYPE`, `LXC_HOOK_SECTION`, `LXC_NAME` 等 |

### 3.4 执行位置（start.c）

```
lxc_spawn()
  ├── pre-start                          [start.c:1035-1037]
  ├── 子进程内：
  │     ├── pre-mount                    [conf.c 内]
  │     ├── mount                        [conf.c:3843-3858]
  │     ├── autodev                      [conf.c 内]
  │     └── start                        [start.c:1495-1499]
  ├── start-host（宿主机侧）              [start.c:2088-2093]
  └── 退出路径：
        ├── stop                          [start.c:1133-1137]
        └── post-stop                     [start.c:1176-1178]
```

## 4. 日志配置（lxc.log.*）

### 4.1 配置项

| 配置键 | 存储字段 | 说明 |
|--------|----------|------|
| `lxc.log.file` | `conf->logfile` | 日志文件路径 |
| `lxc.log.level` | `conf->loglevel` | 日志级别（0-8，0=TRACE） |
| `lxc.log.syslog` | `conf->syslog` | syslog 设施名 |

### 4.2 日志级别

| 数值 | 级别 | 说明 |
|------|------|------|
| 0 | TRACE | 最详细追踪 |
| 1 | DEBUG | 调试信息 |
| 2 | INFO | 一般信息 |
| 3 | NOTICE | 通知 |
| 4 | WARN | 警告 |
| 5 | ERROR | 错误 |
| 6 | CRIT | 严重错误 |
| 7 | ALERT | 紧急警告 |
| 8 | FATAL | 致命错误 |

### 4.3 即时生效

日志配置在解析时即刻生效（`confile.c:1781-1829`）：

```c
set_config_log_file() → lxc_log_set_file()    // 立即切换日志输出
set_config_log_level() → lxc_log_set_level()  // 立即调整级别
```

## 5. 终端配置（lxc.tty.* / lxc.pty.* / lxc.console.*）

### 5.1 TTY 配置

| 配置键 | 默认值 | 说明 |
|--------|--------|------|
| `lxc.tty.max` | 0 | 容器内 TTY 设备数量 |
| `lxc.tty.dir` | "" | TTY 设备挂载子目录 |

**作用**：LXC 会在容器内创建指定数量的 PTY 对，绑定挂载到 `/dev/tty1`、`/dev/tty2` 等。
`tty.dir` 用于指定中间目录，如设置为 `lxc` 则设备先创建在 `/dev/lxc/tty1` 再绑定。

### 5.2 PTY 配置

| 配置键 | 默认值 | 说明 |
|--------|--------|------|
| `lxc.pty.max` | 1024 | 容器内可分配的最大 PTY 数量 |

通过设置 `/dev/pts` 的 `max` 挂载选项实现限制。

### 5.3 Console 配置

| 配置键 | 说明 |
|--------|------|
| `lxc.console.path` | 控制台设备路径（`none` 禁用） |
| `lxc.console.logfile` | 控制台输出日志文件 |
| `lxc.console.rotate` | 日志轮转开关 |
| `lxc.console.buffer.size` | 环形缓冲区大小（字节） |
| `lxc.console.size` | 日志文件最大大小 |

### 5.4 运行时应用

```
terminal.c:624-748   — 控制台 PTY 创建
terminal.c:1048-1137 — 控制台 I/O 转发设置
conf.c:951-976       — 容器内 TTY 设备创建
conf.c:3868-3883     — setup 阶段的终端初始化
```

## 6. 资源限制配置

### 6.1 lxc.prlimit.*

进程资源限制，直接映射到 `setrlimit()` 系统调用。

```ini
lxc.prlimit.nofile = 65536        # 最大打开文件数
lxc.prlimit.nproc = 4096          # 最大进程数
lxc.prlimit.memlock = unlimited   # 锁定内存上限
lxc.prlimit.core = 0              # 核心转储大小
```

**解析**（`confile.c:2100-2174`）：

```
set_config_prlimit(key, value, conf)
  ├── 从 key 提取资源名（"nofile", "nproc" 等）
  ├── 查找 rlimit_map[] 获取 RLIMIT_* 常量
  ├── 解析 soft:hard 格式或单一值
  └── 追加到 conf->limits 链表
```

支持的值格式：
- `N` — soft 和 hard 都设为 N
- `N:M` — soft 设为 N，hard 设为 M
- `unlimited` — 设为 RLIM_INFINITY

**应用**（`conf.c:2994-2996`）：遍历 `conf->limits`，对每项调用 `setrlimit()`。

### 6.2 lxc.sysctl.*

写入内核 sysctl 参数。

```ini
lxc.sysctl.net.ipv4.ip_forward = 1
lxc.sysctl.kernel.shmmax = 268435456
```

**解析**（`confile.c:2176-2225`）：

```
set_config_sysctl(key, value, conf)
  ├── 从 key 提取参数路径（如 "net.ipv4.ip_forward"）
  └── 追加到 conf->sysctls 链表
```

**应用**（`conf.c:2998-3027`）：

```
setup_sysctl_parameters()
  └── 遍历 conf->sysctls：
        ├── 将 '.' 替换为 '/' 构建路径
        │   如 "net.ipv4.ip_forward" → "/proc/sys/net/ipv4/ip_forward"
        └── 写入值到该文件
```

### 6.3 lxc.proc.*

写入 `/proc/<pid>` 下的进程参数。

```ini
lxc.proc.oom_score_adj = -500
```

**解析**（`confile.c:2227-2259`）：与 sysctl 类似，追加到 `conf->procs` 链表。

**应用**（`conf.c:3030-3055`）：写入 `/proc/self/<param>` 文件。

## 7. 启动行为配置（lxc.start.*）

### 7.1 配置项

| 配置键 | 默认值 | 类型 | 说明 |
|--------|--------|------|------|
| `lxc.start.auto` | 0 | bool | 是否随系统自动启动 |
| `lxc.start.delay` | 0 | int | 启动后延迟（秒） |
| `lxc.start.order` | 0 | int | 启动顺序（数字越大越先启动） |

### 7.2 解析

三个键共用 `set_config_start()` 回调（`confile.c:1460-1504`），
通过键名后缀区分设置哪个字段。

### 7.3 lxc-autostart 使用

`lxc-autostart` 工具（`tools/lxc_autostart.c:273-475`）使用这些配置：

```
lxc-autostart
  ├── 扫描所有容器
  ├── 过滤 start.auto == 1 的容器
  ├── 按 start.order 降序排序
  └── 依次启动，每个容器启动后等待 start.delay 秒
```

## 8. 其他配置项

### 8.1 lxc.arch

设置容器的 CPU 架构（personality）：

```ini
lxc.arch = x86_64    # 或 amd64, i686, x86 等
```

通过 `personality()` 系统调用设置进程的执行域。

### 8.2 lxc.uts.name

设置容器的主机名：

```ini
lxc.uts.name = mycontainer
```

通过 `sethostname()` 在 UTS 命名空间中设置。

### 8.3 lxc.no_new_privs

启用 no_new_privs 标志：

```ini
lxc.no_new_privs = 1
```

通过 `prctl(PR_SET_NO_NEW_PRIVS, 1)` 设置，防止通过 exec 获取新权限。

### 8.4 lxc.ephemeral

标记容器为临时容器：

```ini
lxc.ephemeral = 1
```

临时容器在停止后自动销毁。

### 8.5 lxc.keyring.session

控制是否为容器创建新的内核 keyring 会话：

```ini
lxc.keyring.session = 1
```

### 8.6 lxc.sched.core

启用核心调度（Core Scheduling）：

```ini
lxc.sched.core = 1
```

使用 `prctl(PR_SCHED_CORE)` 设置，用于 SMT（超线程）侧信道攻击缓解。

### 8.7 lxc.monitor.*

| 配置键 | 说明 |
|--------|------|
| `lxc.monitor.unshare` | 监控进程是否进入新命名空间 |
| `lxc.monitor.signal.pdeath` | 父进程死亡时发送的信号 |

### 8.8 lxc.time.offset.*

| 配置键 | 说明 |
|--------|------|
| `lxc.time.offset.boot` | boot 时钟偏移量 |
| `lxc.time.offset.monotonic` | monotonic 时钟偏移量 |

用于容器时间虚拟化，需要内核支持时间命名空间（`CLONE_NEWTIME`）。
