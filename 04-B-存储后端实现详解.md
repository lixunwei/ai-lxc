# 存储子系统深度分析：各后端实现

## 1. dir 后端（目录）

**源文件**：`src/lxc/storage/dir.c`  
**特点**：最简单的后端，使用普通目录作为 rootfs。

### 1.1 create

```c
// 创建目录，设置权限
lxc_mkdir_p(dest, 0755);
bdev->src = "dir:<path>";
```

### 1.2 mount

使用 bind mount 将目录挂载到 rootfs mount point：

```c
// 优先使用新 mount API (open_tree + move_mount)
// 回退到传统 mount(src, dest, NULL, MS_BIND|MS_REC, NULL)
// 如果需要只读，追加 MS_REMOUNT|MS_BIND|MS_RDONLY
```

### 1.3 clone

无快照支持，只能全量复制（rsync）。`clone_paths()` 为新容器准备目录路径。

### 1.4 适用场景

- 开发/测试环境
- 不需要快照功能
- 对性能要求不高的场景

---

## 2. btrfs 后端

**源文件**：`src/lxc/storage/btrfs.c`（约 942 行）  
**特点**：利用 btrfs 子卷和快照，性能最优的后端之一。

### 2.1 create

```c
// 使用 ioctl 创建子卷
fd = open(parent_dir);
ioctl(fd, BTRFS_IOC_SUBVOL_CREATE, &args);
// args.name = subvolume_name
```

### 2.2 mount

bind mount 子卷路径（子卷本身已是文件系统的一部分）：

```c
mount(src, dest, "bind", MS_BIND|MS_REC, NULL);
```

### 2.3 snapshot（快照克隆）

```c
// 使用 btrfs 原生快照，瞬间完成（COW）
struct btrfs_ioctl_vol_args_v2 args = {
    .fd = src_fd,
    .flags = 0,  // 可写快照
};
ioctl(dest_fd, BTRFS_IOC_SNAP_CREATE_V2, &args);
```

### 2.4 copy（全量复制）

创建新子卷 + rsync 内容。

### 2.5 destroy

递归删除子卷（btrfs 子卷不能用 rm -rf，需要先删除内嵌子卷）：

```c
// 使用 BTRFS_IOC_TREE_SEARCH 遍历子卷树
// 从叶节点到根逐个 BTRFS_IOC_SNAP_DESTROY_V2
```

### 2.6 适用场景

- 生产环境（高性能 COW）
- 大量容器快照
- 需要高效克隆的 CI/CD 环境

---

## 3. overlay 后端

**源文件**：`src/lxc/storage/overlay.c`（约 700 行）  
**特点**：使用 OverlayFS 分层，下层只读 + 上层可写。

### 3.1 存储结构

```
/var/lib/lxc/myct/
    rootfs/       (lower, 基础镜像，只读)
    delta/        (upper, 增量层，可写)
    work/         (overlayfs 内部使用)
```

### 3.2 create

```c
// 创建 delta 和 work 目录
mkdir(delta_dir);
mkdir(work_dir);
// src 格式: "overlay:<lower>:<upper>"
bdev->src = "overlay:/path/lower:/path/delta";
```

### 3.3 mount

```c
// 构造 overlayfs mount options
char options[PATH_MAX];
snprintf(options, "lowerdir=%s,upperdir=%s,workdir=%s", lower, upper, work);
mount("overlay", dest, "overlay", 0, options);

// 兼容旧内核：尝试 "overlayfs" 类型名
// 兼容更旧内核：省略 workdir 选项
```

### 3.4 clone（快照模式）

overlay 的"快照"是将当前 rootfs 作为新的 lower layer：

```c
// 原容器 rootfs 变为新容器的 lower
// 新容器获得空的 delta 层
// 如果原容器也是 overlay，lower 层叠加
```

### 3.5 适用场景

- 基于共享基础镜像的多容器部署
- 模板化容器（golden image + per-container delta）
- 磁盘空间受限环境

---

## 4. loop 后端

**源文件**：`src/lxc/storage/loop.c`（约 322 行）  
**特点**：使用 raw 镜像文件 + loop 设备。

### 4.1 create

```c
// 1. 创建稀疏文件
fd = open(path, O_CREAT|O_RDWR);
lseek(fd, size - 1, SEEK_SET);
write(fd, "\0", 1);

// 2. 格式化文件系统
do_mkfs_exec_wrapper(path, fstype);  // fork + exec mkfs.ext4/xfs/...
```

### 4.2 mount

```c
// 1. 分配 loop 设备
lofd = lxc_prepare_loop_dev(src, LO_FLAGS_AUTOCLEAR);
// ioctl(loopfd, LOOP_SET_FD, filefd)
// ioctl(loopfd, LOOP_SET_STATUS64, &info)

// 2. 挂载 loop 设备
mount_unknown_fs("/dev/loopN", dest, mntopts);
// 遍历 /proc/filesystems 尝试各种 fstype
```

### 4.3 特性

- `LO_FLAGS_AUTOCLEAR`：容器停止后自动释放 loop 设备
- 支持任意文件系统类型（ext4、xfs、btrfs 等）
- 文件大小固定（创建时指定）

### 4.4 适用场景

- 需要文件系统隔离（quota、inode 限制）
- 可移植的容器镜像
- 测试不同文件系统类型

---

## 5. zfs 后端

**源文件**：`src/lxc/storage/zfs.c`（约 758 行）  
**特点**：使用 ZFS dataset，支持快照和克隆。

### 5.1 create

```c
// 通过 zfs CLI 创建 dataset
run_command("zfs create -o mountpoint=<path> -o canmount=noauto -p <dataset>");
// 例: zfs create -o mountpoint=/var/lib/lxc/ct/rootfs lxc/containers/ct
```

### 5.2 mount

```c
// 直接挂载 ZFS dataset
mount(dataset, dest, "zfs", 0, "zfsutil,mntpoint=...");

// legacy 模式：通过 `zfs list` 查找 dataset 名称
```

### 5.3 snapshot

```c
// 1. 创建递归快照
run_command("zfs snapshot -r <dataset>@<snapname>");
// 2. 从快照克隆
run_command("zfs clone -p <dataset>@<snap> <new_dataset>");
```

### 5.4 copy（全量复制）

```c
// 创建新 dataset + rsync
run_command("zfs create ...");
rsync(old_mount, new_mount);
```

### 5.5 destroy

```c
// 删除 dataset 及其快照
run_command("zfs destroy -r <dataset>");
// 清理父 snapshot（如果是克隆源）
```

### 5.6 适用场景

- 企业级存储需求
- 需要数据完整性校验
- 高效快照和克隆
- 存储压缩/去重

---

## 6. lvm 后端

**源文件**：`src/lxc/storage/lvm.c`（约 657 行）  
**特点**：使用 LVM 逻辑卷，支持快照（thin/thick）。

### 6.1 create

```c
// 普通 LV
run_command("lvcreate -L <size> -n <lv_name> <vg>");

// Thin LV (如果指定了 thinpool)
run_command("lvcreate --thinpool <pool> -V <size> -n <lv_name> <vg>");

// 格式化
do_mkfs_exec_wrapper("/dev/<vg>/<lv>", fstype);
```

### 6.2 mount

```c
// 探测文件系统类型并挂载
mount_unknown_fs("/dev/vg/lv", dest, mntopts);
```

### 6.3 snapshot

```c
// Thick snapshot
run_command("lvcreate -s -L <size> -n <snap_name> /dev/<vg>/<orig_lv>");

// Thin snapshot (无需指定大小)
run_command("lvcreate -s -n <snap_name> /dev/<vg>/<orig_lv>");

// XFS/Btrfs UUID 重新生成（避免冲突）
run_command("xfs_admin -U generate /dev/<vg>/<snap>");
run_command("btrfstune -f -u /dev/<vg>/<snap>");
```

### 6.4 copy（全量复制）

```c
// 创建新 LV + mkfs + rsync
lvcreate(...);
mkfs(...);
mount -> rsync -> umount;
```

### 6.5 特性

- Thin provisioning 支持（按需分配空间）
- LV 属性检查通过 `lvs --noheadings -o lv_attr`
- 签名擦除：`lvcreate --wipesignatures y`（如支持）

### 6.6 适用场景

- 传统企业环境
- 需要块级快照
- 与现有 LVM 基础设施集成

---

## 7. nbd 后端

**源文件**：`src/lxc/storage/nbd.c`（约 324 行）  
**特点**：通过 qemu-nbd 连接网络块设备或 qcow2 镜像。

### 7.1 create

不支持（返回 -ENOSYS）。需要外部准备镜像。

### 7.2 mount

```c
// 1. 使用 qemu-nbd 连接镜像
run_command("qemu-nbd -c /dev/nbdN <image_path>");

// 2. 挂载块设备（可能有分区 /dev/nbdNpM）
mount_unknown_fs("/dev/nbdN", dest, mntopts);
```

### 7.3 umount

```c
umount(dest);
run_command("qemu-nbd -d /dev/nbdN");
```

### 7.4 特性

- 支持 qcow2、raw、vmdk 等格式（qemu-nbd 支持的所有格式）
- 使用 signalfd + PDEATHSIG 确保父进程死亡时自动断开
- 分区检测（/dev/nbdNpM）

### 7.5 适用场景

- 使用 VM 磁盘镜像运行容器
- qcow2 快照（通过 qemu-nbd）
- 远程块设备

---

## 8. rbd 后端

**源文件**：`src/lxc/storage/rbd.c`（约 282 行）  
**特点**：使用 Ceph RBD（RADOS Block Device）。

### 8.1 create

```c
// 1. 创建 RBD image
run_command("rbd create --pool <pool> --size <size> <image>");

// 2. 映射到本地
run_command("rbd map --pool <pool> <image>");
// 获得 /dev/rbd/<pool>/<image>

// 3. 格式化
do_mkfs_exec_wrapper("/dev/rbd/pool/image", fstype);
```

### 8.2 mount

```c
// 挂载已映射的块设备
mount_unknown_fs("/dev/rbd/pool/image", dest, mntopts);
```

### 8.3 destroy

```c
run_command("rbd unmap /dev/rbd/pool/image");
run_command("rbd rm --pool <pool> <image>");
```

### 8.4 适用场景

- Ceph 分布式存储集群
- 需要远程块存储
- 大规模容器部署

## 9. 后端对比总结

| 后端 | 存储类型 | 快照速度 | 空间效率 | 复杂度 | 生产就绪 |
|------|----------|----------|----------|--------|----------|
| dir | 目录 | N/A | 低（全量） | ★☆☆ | ✅ |
| btrfs | 子卷 | O(1) COW | 高 | ★★☆ | ✅ |
| overlay | 分层 | O(1)（新层）| 高 | ★★☆ | ✅ |
| loop | 文件 | N/A | 中（预分配） | ★★☆ | ✅ |
| zfs | Dataset | O(1) COW | 高 | ★★★ | ✅ |
| lvm | 逻辑卷 | O(1)（thin）| 中-高 | ★★★ | ✅ |
| nbd | 网络块 | 取决于后端 | 取决于格式 | ★★★ | ⚠️ |
| rbd | Ceph块 | N/A | 分布式 | ★★★ | ⚠️ |

## 10. rsync 辅助（storage/rsync.c）

全量复制统一使用 rsync：

```c
// rsync 参数
char *const args[] = {
    "rsync",
    "-aHXS",          // archive, hardlinks, xattrs, sparse
    "--devices",
    "--delete",
    src, dest,
    NULL,
};
```

用于：
- dir 后端的 clone
- btrfs/zfs/lvm 的 full copy 路径
- overlay 的 delta 层复制
