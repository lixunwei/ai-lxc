# 03-C 网络子系统深入分析：lxc-user-nic 与网络配置项

## 1. lxc-user-nic 辅助程序

### 1.1 定位与安全模型

`lxc-user-nic` 是一个 **setuid-root** 的辅助程序，用于为非特权容器创建和管理网络设备。

**核心问题**：非特权用户无法创建 veth pair 或将接口绑定到桥上，因此需要一个特权辅助程序代为执行这些操作。

**安全模型**：
- 程序启动后立即**清空所有环境变量**，重设 `PATH`（`lxc_user_nic.c:1110-1121`）
- 只接受两个命令：`create` 和 `delete`
- 严格验证调用者权限
- 通过 `/etc/lxc/lxc-usernet` 控制配额

### 1.2 支持的操作

| 命令 | 功能 | 权限检查 |
|------|------|----------|
| `create` | 创建 veth pair 并绑定到桥 | 调用者必须能访问目标 netns |
| `delete` | 删除网络接口（仅 OVS） | 调用者必须对目标 netns 有特权 |

### 1.3 usernet 配置文件

**路径**：`/etc/lxc/lxc-usernet`

**格式**：
```
# 用户名    类型    桥名      数量
user1      veth    lxcbr0    10
@group1    veth    br0       5
```

| 字段 | 说明 |
|------|------|
| 用户/组 | 用户名或 `@组名` |
| 类型 | 网络类型（如 `veth`） |
| 桥名 | 目标桥接口名 |
| 数量 | 该用户/组允许创建的最大接口数 |

**解析逻辑**（`get_alloted()`，`lxc_user_nic.c:309-355`）：
1. 逐行 `getline()` + `sscanf()` 解析
2. 用户名必须匹配当前用户，或 `@组名` 匹配用户所属组
3. `type` 必须等于请求的类型
4. `bridge` 必须等于请求的桥名
5. 匹配条目的 `count` 累加为总配额

### 1.4 创建 veth 流程

```
create 命令流程
  │
  ├── 1. 安全初始化
  │     ├── 清空环境变量，重设 PATH
  │     ├── getpwuid_r() 获取当前用户信息
  │     └── lxc_safe_int() 严格解析 PID
  │
  ├── 2. 配额检查
  │     ├── 解析 /etc/lxc/lxc-usernet → 计算 allowed
  │     ├── 清理数据库中无效条目
  │     ├── count_entries() 统计已使用数量
  │     └── count >= allowed? → 拒绝
  │
  ├── 3. 权限检查
  │     └── may_access_netns(pid)
  │         ├── 降权到调用者 UID
  │         ├── access("/proc/<pid>/ns/net", R_OK)
  │         └── 恢复权限
  │
  ├── 4. 生成接口名
  │     ├── 宿主端: veth<uid>_XXXX
  │     └── 容器端: <name>p
  │
  ├── 5. 创建 veth pair
  │     └── instantiate_veth()
  │         ├── lxc_veth_create(veth1, veth2, pid, mtu, -1, -1)
  │         ├── setup_private_host_hw_addr(veth1)  ← 修改宿主端 MAC
  │         └── netdev_set_flag(veth1, IFF_UP)     ← 启用宿主端
  │
  ├── 6. 获取桥的 MTU 并设置
  │     └── netdev_get_mtu(bridge_ifindex) → lxc_netdev_set_mtu()
  │
  ├── 7. 绑定到桥
  │     └── lxc_bridge_attach(bridge, veth1)
  │         ├── ioctl(SIOCBRADDIF)  ← 普通桥
  │         └── ovs-vsctl add-port  ← OVS 桥
  │
  ├── 8. 在容器 netns 中重命名
  │     └── lxc_secure_rename_in_ns()
  │         └── 安全地进入目标 netns 重命名接口
  │
  ├── 9. 记录到数据库
  │     └── 追加 "user type bridge nicname" 到 LXC_USERNIC_DB
  │
  └── 10. 失败回滚
        └── 创建失败时删除已建 veth
```

### 1.5 配额管理

```
配额数据库 (LXC_USERNIC_DB)
  │
  ├── 格式: user type bridge nicname
  │   例: user1 veth lxcbr0 veth1001_ab3c
  │
  ├── 清理无效条目:
  │   ├── 检查对应的网络接口是否仍存在
  │   └── 不存在 → 从数据库中移除
  │
  ├── 计数:
  │   count_entries() 统计当前用户在指定 type+bridge 下的条目数
  │
  └── 创建时:
      ├── count < allowed → 允许创建，追加条目
      └── count >= allowed → 拒绝
```

### 1.6 安全检查清单

| 检查项 | 位置 | 说明 |
|--------|------|------|
| 清空环境变量 | `1110-1121` | 防止环境变量注入 |
| 用户名解析 | `107-133` | `getpwuid_r()` 安全获取 |
| PID 严格解析 | `1129-1134` | `lxc_safe_int()` 防止溢出 |
| netns 文件类型 | `1141-1168` | 检查 `/proc/<pid>/ns/net` 是否为 `NSFS_MAGIC` |
| 访问权限（create） | `945-987` | 降权后检查 `access(R_OK)` |
| 特权检查（delete） | `1004-1068` | 在目标 netns 内尝试删除 lo |
| 接口名合法性 | `561-571` | `IFNAMSIZ` 长度 + 字符检查 |
| 失败回滚 | `609-611` | 创建失败时清理已建资源 |
| CVE-2022-47952 | `1138-1145` | 防止 open 成功/失败信息泄露 |

### 1.7 delete 命令

```
delete 命令流程
  │
  ├── 1. 验证 netns 文件类型
  │
  ├── 2. 特权检查: is_privileged_over_netns(netns_fd)
  │     ├── setns() 进入目标 netns
  │     ├── 尝试删除 lo 接口
  │     └── 返回 ENOTSUP → 视为有特权
  │
  ├── 3. 仅处理 OVS bridge
  │     ├── is_ovs_bridge(link)?
  │     │   ├── [是] lxc_ovs_delete_port(link, veth_name)
  │     │   └── [否] 跳过（普通桥的接口随 netns 自动清理）
  │
  └── 4. 从数据库中移除条目
```

### 2.2 配置示例

#### veth + 桥接模式

```ini
lxc.net.0.type = veth
lxc.net.0.link = lxcbr0
lxc.net.0.flags = up
lxc.net.0.name = eth0
lxc.net.0.hwaddr = 00:16:3e:xx:xx:xx
lxc.net.0.ipv4.address = 10.0.3.100/24
lxc.net.0.ipv4.gateway = 10.0.3.1
lxc.net.0.ipv6.address = fd00::100/64
lxc.net.0.ipv6.gateway = fd00::1
```

#### veth + 路由模式

```ini
lxc.net.0.type = veth
lxc.net.0.veth.mode = router
lxc.net.0.flags = up
lxc.net.0.name = eth0
lxc.net.0.ipv4.address = 10.0.3.100/32
lxc.net.0.ipv4.gateway = auto
lxc.net.0.veth.ipv4.route = 10.0.3.100/32
```

#### macvlan

```ini
lxc.net.0.type = macvlan
lxc.net.0.macvlan.mode = bridge
lxc.net.0.link = eth0
lxc.net.0.flags = up
lxc.net.0.name = eth0
lxc.net.0.ipv4.address = 192.168.1.100/24
lxc.net.0.ipv4.gateway = 192.168.1.1
```

#### 多网卡

```ini
# 第一块网卡
lxc.net.0.type = veth
lxc.net.0.link = lxcbr0
lxc.net.0.name = eth0
lxc.net.0.ipv4.address = 10.0.3.100/24

# 第二块网卡
lxc.net.1.type = macvlan
lxc.net.1.macvlan.mode = bridge
lxc.net.1.link = eth0
lxc.net.1.name = eth1
lxc.net.1.ipv4.address = 192.168.1.100/24
```

#### 仅 loopback

```ini
lxc.net.0.type = empty
```

## 3. 网络子系统函数索引

### 3.1 按功能分类

#### 接口生命周期

| 函数 | 功能 |
|------|------|
| `lxc_create_network()` | 创建所有配置的网络设备 |
| `lxc_delete_network()` | 删除所有网络设备 |
| `lxc_setup_network_in_child_namespaces()` | 在容器 namespace 中配置网络 |

#### 接口创建

| 函数 | 功能 |
|------|------|
| `lxc_veth_create()` | 创建 veth pair |
| `lxc_macvlan_create()` | 创建 macvlan |
| `lxc_vlan_create()` | 创建 VLAN 子接口 |

#### 接口操作

| 函数 | 功能 |
|------|------|
| `lxc_netdev_up()` / `lxc_netdev_down()` | 启用/关闭接口 |
| `lxc_netdev_set_mtu()` | 设置 MTU |
| `lxc_netdev_rename_by_index()` / `_by_name()` | 重命名接口 |
| `lxc_netdev_delete_by_index()` / `_by_name()` | 删除接口 |
| `lxc_netdev_move_by_index()` / `_by_name()` | 移动接口到其他 netns |
| `lxc_netdev_move_wlan()` | 移动 WLAN 接口 |

#### 地址与路由

| 函数 | 功能 |
|------|------|
| `lxc_ipv4_addr_add()` / `lxc_ipv6_addr_add()` | 添加 IP 地址 |
| `lxc_ipv4_addr_get()` / `lxc_ipv6_addr_get()` | 获取 IP 地址 |
| `lxc_ipv4_gateway_add()` / `lxc_ipv6_gateway_add()` | 添加默认网关 |
| `lxc_ip_route_dest()` | 添加/删除目标路由 |

#### 桥接

| 函数 | 功能 |
|------|------|
| `lxc_bridge_attach()` | 将接口挂接到桥 |
| `is_ovs_bridge()` | 检测是否为 OVS 桥 |
| `lxc_ovs_delete_port()` | 删除 OVS 端口 |

#### 代理与转发

| 函数 | 功能 |
|------|------|
| `lxc_ip_forwarding_on()` / `_off()` | 开启/关闭 IP 转发 |
| `lxc_neigh_proxy_on()` / `_off()` | 开启/关闭邻居代理 |
| `lxc_ip_neigh_proxy()` | 添加/删除邻居代理条目 |

#### 父子进程协调

| 函数 | 功能 |
|------|------|
| `lxc_network_send_to_child()` | 向子进程发送网络信息 |
| `lxc_network_recv_from_parent()` | 从父进程接收网络信息 |
| `lxc_network_send_name_and_ifindex_to_parent()` | 回传最终接口名和 ifindex |
| `lxc_network_recv_name_and_ifindex_from_child()` | 接收子进程反馈 |

#### 工具函数

| 函数 | 功能 |
|------|------|
| `lxc_convert_mac()` | MAC 字符串转 sockaddr |
| `lxc_ifname_alnum_case_sensitive()` | 基于模板生成唯一接口名 |
| `lxc_net_type_to_str()` | 网络类型枚举转字符串 |
| `setup_private_host_hw_addr()` | 设置宿主端私有 MAC |
| `netdev_get_mtu()` | 获取接口 MTU |
| `is_wlan()` | 检测是否为 WLAN 接口 |
| `lxc_netns_set_nsid()` / `lxc_netns_get_nsid()` | 设置/获取 netns ID |
